Description

The application header is a persistent layout component that provides branding, primary navigation controls (like the mobile menu toggle), and user-specific actions such as notifications and profile management. This variant is specifically for the Admin web dashboard.

Design System Alignment

This component is an organism composed of several atomic and molecular components. It uses the design system's color palette for background and text, spacing for padding and layout, shadows for elevation, and typography. It directly integrates the Button, Icon, Avatar, and Dropdown components.

Responsive Specifications

Mobile Adaptations

  • The hamburger menu toggle button is displayed on the left.
  • The user profile details (name, role) are hidden, showing only the avatar and chevron.

Tablet Adaptations

  • The hamburger menu toggle remains visible.
  • The user profile details (name, role) are displayed to utilize the extra space.

Desktop Enhancements

  • The hamburger menu toggle is hidden, as the full SidebarNav is expected to be visible.
  • All elements are fully displayed with optimized spacing for a desktop viewport.

Accessibility Features

Keyboard Support

All interactive elements (menu toggle, notification bell, user profile trigger, dropdown items) are focusable and activatable using the keyboard.

Screen Reader Support

The hamburger menu toggle and notification bell use `aria-label` for a descriptive name. The user profile dropdown trigger uses `aria-haspopup` and `aria-expanded` to announce its state.

Focus Management

All interactive elements have a clear `:focus-visible` state. JavaScript would be required to trap focus within the user profile dropdown when it's open.

Aria Attributes

  • aria-label
  • aria-haspopup="menu"
  • aria-expanded="true/false"

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast requirements against their backgrounds.
