Responsive Specifications

Desktop Layout

  • Two-column layout with the sidebar on the left and main content on the right.
  • Sidebar can be in an 'expanded' state (with icons and text) or 'collapsed' state (icons only).

Tablet Adaptations

  • Sidebar automatically defaults to the 'collapsed' (icon-only) state to maximize content area.

Mobile Adaptations

  • Sidebar is hidden by default (off-canvas).
  • A header with a hamburger menu icon is displayed.
  • Clicking the hamburger icon slides the sidebar into view over the main content.

Accessibility Features

Keyboard Navigation

Complete tab order through all navigation links. The hamburger menu is keyboard-operable.

Screen Reader Support

The sidebar is contained within a `

` landmark. Each link has descriptive text. In collapsed mode, `aria-label` or visually hidden text ensures links are understood.

Focus Management

A clear focus-visible state is applied to all navigation links. When the mobile menu is open, focus is trapped within it.

Aria Attributes

  • aria-label for icon-only links in collapsed mode
  • aria-current='page' for the active navigation link
  • aria-expanded and aria-controls for the mobile menu toggle

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast ratios against the background.
