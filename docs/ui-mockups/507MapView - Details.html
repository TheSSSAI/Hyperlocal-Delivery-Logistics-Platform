Description

A highly specialized and interactive component for displaying geographical data. It supports different variants for live delivery tracking, interactive address selection with a map pin, and a polygon drawing tool for administrative geofencing.

Design System Alignment

This organism combines multiple atomic and molecular components such as Icon, Spinner, Button, Card, and FormGroup. It leverages the design system's tokens for color, spacing, shadows, and typography to create a consistent and branded user experience for all map-related interactions.

Responsive Specifications

Mobile Adaptations

  • Map interactions are optimized for touch gestures (pinch-to-zoom, pan).
  • Overlays like the geofencing tool control panel would typically be presented in a bottom sheet or a full-screen modal to be usable on small screens.

Tablet Adaptations

  • Maintains the same touch-friendly interactions as mobile.

Desktop Enhancements

  • Supports mouse-based interactions (click-and-drag, scroll-to-zoom).
  • Overlays can be positioned as side panels (as shown in the geofencing mockup) for better use of screen real estate.

Accessibility Features

Keyboard Support

The map container should be keyboard focusable (`tabindex='0'`) to allow keyboard users to pan the map with arrow keys. All interactive controls (buttons, inputs) within overlays must be keyboard navigable.

Screen Reader Support

The map view should have an `aria-label` describing its purpose (e.g., 'Live order tracking map'). Changes in map state (e.g., rider location updates) should be announced via an ARIA live region.

Focus Management

When an overlay or modal (like the geofencing controls) is opened, focus should be trapped within it.

Aria Attributes

  • aria-label for the map container
  • aria-live for real-time update regions
  • role='dialog'` and `aria-modal='true'` for modal overlays

Color Contrast

Text and interactive elements within overlays meet WCAG 2.1 AA contrast requirements.
