Description

A molecular component for entering One-Time Passwords (OTPs). It consists of a group of individual input fields, one for each digit, to provide a clear and touch-friendly user experience. It supports different lengths and visual states like default, focus, filled, and error.

Design System Alignment

This component is built upon the atomic 'TextInput' component's styling principles. It uses the design system's tokens for borders, colors (primary for focus, error for invalid states), spacing, typography, and shadows to ensure visual consistency with other form elements.

Responsive Specifications

Mobile Adaptations

  • Input segments are touch-friendly with a minimum height of 48px.
  • On smaller screens (e.g., < 480px), the gap between inputs is reduced and segment size is slightly smaller to fit within narrow viewports.

Tablet Adaptations

  • Maintains the default desktop sizing, which is already touch-friendly.

Desktop Enhancements

  • Clear focus states for keyboard navigation.
  • Sufficient spacing for a clean layout.

Accessibility Features

Keyboard Support

Standard keyboard support for text inputs. JavaScript is required to auto-tab between inputs upon entry and backspace.

Screen Reader Support

The set of inputs is wrapped in a `fieldset` with a `legend` for context (e.g., 'Verification Code'). Each input has an `aria-label` like 'Digit 1 of 6'.

Focus Management

A clear, high-contrast focus ring is applied to the active input segment. JavaScript is needed to move focus automatically as the user types.

Aria Attributes

  • aria-label for each digit input.
  • aria-describedby to link to error messages.

Color Contrast

Input borders and text meet WCAG 2.1 AA contrast requirements in all states.
