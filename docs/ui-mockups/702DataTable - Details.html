Description

A responsive data table for displaying sets of data. It supports sorting, pagination, and various states including loading, empty, and with-data. On mobile, it transforms into a list of cards for better readability.

Design System Alignment

This component is composed of several atomic components including Spinner and Button (for Pagination). It utilizes the design system's tokens for colors, typography, spacing, borders, and shadows to ensure a cohesive look and feel with the rest of the application.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 768px, the table transforms into a list of cards.
  • The `` is hidden.
  • Each `` becomes a block-level element with its own border and padding, resembling a card.
  • Each `` stacks vertically within the 'card' and uses a `::before` pseudo-element with a `data-label` attribute to display its corresponding column header.
  • This ensures data remains readable and accessible on narrow viewports without requiring horizontal scrolling.

Tablet Adaptations

  • The standard horizontal table layout is maintained, providing a data-dense view suitable for wider screens.

Desktop Enhancements

  • The horizontal table layout is optimal for desktop, allowing for easy scanning of rows and columns.
  • Hover states on rows provide clear visual feedback.
  • Sticky table headers could be added for very long tables to keep column context visible while scrolling.

Accessibility Features

Keyboard Support

The table itself is not interactive, but interactive elements within it (like sortable headers, buttons, pagination) are focusable via the Tab key and can be activated with Enter/Space.

Screen Reader Support

Uses semantic `

Focus Management

Interactive elements within the table have clear `:focus-visible` styles as defined in the atomic component layer.

Aria Attributes

  • On mobile, the `data-label` provides a visible label for each piece of data, which is inherently accessible.
  • The loading state wrapper uses `aria-busy="true"` to inform screen readers that content is updating.

Color Contrast

All text and background color combinations meet WCAG 2.1 AA contrast requirements. Status indicators (badges) use both color and text to convey information.
