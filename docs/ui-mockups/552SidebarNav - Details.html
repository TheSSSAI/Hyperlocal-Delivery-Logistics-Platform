Description

A responsive sidebar navigation component for the Vendor and Admin web dashboards. It provides primary navigation links with icons and text, supports expanded and collapsed states, and is designed to be fully accessible and responsive.

Design System Alignment

This component is an organism that composes atomic Link and Icon components. It heavily utilizes the design system's tokens for colors (primary, neutrals), spacing, typography, and responsive breakpoints to ensure visual consistency.

Responsive Specifications

Mobile Adaptations

  • The sidebar is hidden off-canvas by default on screens narrower than 768px.
  • It is intended to be revealed by a toggle button in the AppHeader, which would add a class like `.sidebar-nav--mobile-open`.

Tablet Adaptations

  • On screens between 768px and 1024px, the sidebar defaults to the collapsed (icon-only) state to maximize content area.

Desktop Enhancements

  • The sidebar is fully expanded by default on screens wider than 1024px, showing both icons and text labels.
  • Hover effects are enabled for richer interaction feedback.

Accessibility Features

Keyboard Support

All navigation links are keyboard-focusable using the Tab key and can be activated with Enter.

Screen Reader Support

The component is wrapped in a `

` element with an `aria-label`. The active link is identified using `aria-current='page'`. In the collapsed state, links should have `aria-label`s that match the full text label for context.

Focus Management

A clear, visible focus style is applied to links when they receive keyboard focus, ensuring navigability without a mouse.

Aria Attributes

  • aria-label on the main
    element.
  • aria-current='page' for the active navigation link.
  • aria-hidden='true' for decorative icons.

Color Contrast

Text and icon colors meet WCAG 2.1 AA contrast requirements against the dark background. The active state also maintains sufficient contrast.
