Description

A self-contained organism for real-time, two-way communication within the context of a specific order. It includes a message history, a text input area with a send button, and optional quick-reply buttons. The component supports various states such as active, read-only, loading, and error.

Design System Alignment

This component is an organism composed of several molecular and atomic components from the design system, including TextInput, Button, Icon, Spinner, and custom-defined MessageBubble and QuickReply components. It adheres to the established visual foundation for typography, color, spacing, and borders.

Responsive Specifications

Mobile Adaptations

  • The component is designed mobile-first, intended to take up the full screen height on mobile devices.
  • The layout adapts to the on-screen keyboard by shrinking the message area, which is a native OS behavior handled by the parent layout.

Tablet Adaptations

  • The component can be used within a modal or a dedicated panel in a multi-column layout on tablets, scaling to its container.

Desktop Enhancements

  • The component is well-suited for use in a side panel or modal within the Admin or Vendor dashboards for support and order management.

Accessibility Features

Keyboard Support

The quick-reply buttons, text input, and send button are all keyboard focusable and can be activated with Enter/Space.

Screen Reader Support

The message area can be given `role="log"` and `aria-live="polite"` to announce new messages. Buttons have `aria-label` where necessary. The entire component should be labeled with `aria-labelledby` pointing to the header title.

Focus Management

Focus is managed within the input and button elements as per standard browser behavior.

Aria Attributes

  • role="log"
  • aria-live="polite"
  • aria-labelledby

Color Contrast

All text and background color combinations for message bubbles, buttons, and text meet WCAG 2.1 AA contrast requirements.
