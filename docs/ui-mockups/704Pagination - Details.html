Description

A navigation component for traversing paginated data. It's composed of buttons and links to navigate between pages. It adapts responsively for mobile screens.

Design System Alignment

This component is built from atomic Button and Icon components. It uses design system tokens for colors (primary for active state, neutrals for others), spacing for layout, and typography for text elements. The overall look and feel align with other form and navigation components.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 768px, the numbered page links and ellipsis are hidden.
  • Only the 'Previous' and 'Next' buttons are displayed to save space.
  • An optional text element (e.g., 'Page 3 of 10') can be shown between the buttons for context.

Tablet Adaptations

  • The full pagination component is displayed, same as on desktop.

Desktop Enhancements

  • Displays the full component with 'Previous' and 'Next' buttons, a set of numbered page links, and ellipsis (...) for truncated page ranges.
  • Includes hover states for interactive elements.

Accessibility Features

Keyboard Support

The component is fully keyboard-navigable using the Tab key. Page links are activated with Enter/Space.

Screen Reader Support

The component is wrapped in a `

` element with `aria-label="pagination"`. The currently active page is identified with `aria-current="page"`. Disabled buttons have `aria-disabled="true"`.

Focus Management

Each interactive link/button has a clear `:focus-visible` state that meets WCAG 2.1 AA requirements for visibility.

Aria Attributes

  • aria-label="pagination"
  • aria-current="page"
  • aria-disabled="true"

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast requirements in all states (default, active, disabled).
