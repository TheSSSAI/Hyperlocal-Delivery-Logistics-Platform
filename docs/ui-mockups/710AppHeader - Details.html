Description

The AppHeader is a responsive, high-level layout component that provides consistent branding, navigation, and user-centric actions across all four platform applications. It adapts its content and layout based on the user role (Customer, Rider, Vendor, Admin) and viewport size.

Design System Alignment

This organism is composed of multiple atomic and molecular components, including Button, Icon, Link, SearchBar, and Avatar. It heavily relies on the design system's color palette for backgrounds and text, spacing tokens for layout, and typography for all text elements. Its responsive behavior is a key implementation of the system's responsive strategy.

Responsive Specifications

Mobile Adaptations

  • For web dashboards (Vendor, Admin), the main navigation links collapse into a hamburger menu.
  • The search bar collapses into an icon-only button to save space.
  • The mobile-native headers (Customer, Rider) have a simplified, fixed layout.

Tablet Adaptations

  • The web dashboards will show the mobile (hamburger) menu on portrait tablets and may show the full desktop navigation on landscape tablets.

Desktop Enhancements

  • Web dashboards display the full horizontal navigation with text links.
  • The SearchBar component is fully expanded.
  • Hover states are active for interactive elements like navigation links and user profile avatars.

Accessibility Features

Keyboard Support

All interactive elements (links, buttons, user menu) are keyboard focusable in a logical order.

Screen Reader Support

Uses semantic `

` and `
` elements. Icon-only buttons (like the hamburger menu and notification bell) have descriptive `aria-label`s. The hamburger button uses `aria-expanded` to indicate the state of the mobile menu.

Focus Management

When the mobile menu is opened (requires JS), focus is trapped within the menu panel to prevent users from accidentally tabbing to the background content.

Aria Attributes

  • aria-label for icon buttons
  • aria-expanded for the mobile menu toggle
  • aria-controls to link the toggle to the menu panel

Color Contrast

All text and icons meet WCAG 2.1 AA contrast requirements against the header background.
