Description

A specialized organism component for administrators to create, view, edit, and delete operational zones using an interactive map and a data table. It combines a map view, a data table for listing zones, and a form group for editing zone details.

Design System Alignment

This component is composed of molecular components (DataTable, FormGroup) and atomic components (Button), all of which are styled using the foundational design system tokens for color, typography, spacing, and shadows.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 1024px, the layout stacks vertically, with the sidebar/list on top and the map below.
  • The interface is primarily desktop-focused, so complex drawing actions may be difficult on mobile.

Tablet Adaptations

  • The stacked layout provides a usable interface for viewing and simple edits.

Desktop Enhancements

  • The two-column layout provides an optimal experience, allowing the user to see the list/form and the map simultaneously.

Accessibility Features

Keyboard Support

All interactive elements in the sidebar (buttons, table rows, form fields) are keyboard-focusable and operable.

Screen Reader Support

The DataTable uses semantic `

` elements. Buttons and form fields have proper labels. Map interactions would require JavaScript to provide accessible alternatives or descriptions.

Focus Management

Visible focus states are applied to all interactive elements. JavaScript would be required to manage focus when modals or different states are activated.

Aria Attributes

  • aria-label for buttons and controls.
  • aria-selected on table rows.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
