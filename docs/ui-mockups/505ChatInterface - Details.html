Description

A full-featured chat interface for real-time communication between users. It includes a message history, a text input area with a send button, quick reply options, and handles different states like active, read-only, loading, and error.

Design System Alignment

This organism combines molecular components like TextInput and Button, and atomic components like Icons and Spinners. It heavily relies on the design system's tokens for color, typography, spacing, and shadows to create a cohesive and branded communication experience.

Responsive Specifications

Mobile Adaptations

  • The component is designed mobile-first, occupying the full height of its container.
  • The input footer is designed to sit cleanly above the virtual keyboard when it appears.
  • Quick replies are horizontally scrollable to accommodate many options on a narrow screen.

Tablet Adaptations

  • The interface scales gracefully to tablet sizes without layout changes.

Desktop Enhancements

  • The interface works well in a column or modal on desktop dashboards, maintaining its mobile-first structure.

Accessibility Features

Keyboard Support

The input field is focusable, and messages can be sent by activating the 'Send' button with Enter/Space. Quick replies are focusable buttons.

Screen Reader Support

The message list has `aria-live='polite'` to announce new incoming messages. The input has a hidden label. The send button and quick replies have descriptive `aria-label`s.

Focus Management

Focus is managed within the chat input area. All interactive elements have clear focus indicators.

Aria Attributes

  • aria-live='polite'
  • aria-label for the main input and buttons

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast ratio requirements.
