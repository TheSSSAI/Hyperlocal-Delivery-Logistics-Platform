Component Variations

Variant Name : Default State

This is the standard appearance of the component for a single time range. The first time slot cannot be removed to ensure at least one entry always exists.

Variant Name : Multi-Slot State

Demonstrates the component with multiple time slots for a single day, such as for a business with a lunch break. Subsequent time slots can be removed.

Variant Name : Error State

This state is activated when there's a validation error, such as overlapping time slots or a closing time that is before the opening time. The legend and input borders turn red, and a descriptive error message is displayed.
