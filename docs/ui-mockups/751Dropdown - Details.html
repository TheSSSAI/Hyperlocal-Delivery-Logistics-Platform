Description

A custom select component that allows users to choose one or more options from a list. It combines a trigger element with a dropdown panel containing list items.

Design System Alignment

This component utilizes the design system's color palette for states, spacing for padding, border-radius for shape, and typography for text. It's built to resemble the atomic TextInput for visual consistency in forms.

Responsive Specifications

Mobile Adaptations

  • The component is designed to be responsive and will take the full width of its container on smaller screens.
  • For optimal usability on mobile, this custom component should be replaced by the native on mobile for usability, which would be a JS-driven enhancement.

Accessibility Compliance

The HTML structure and ARIA attributes are designed for WCAG 2.1 AA compliance. Full compliance requires JavaScript to manage states and focus.

Browser Support

Designed for modern browsers. Uses standard CSS properties that have wide support.

Usage Guidelines

When To Use

Use when a user needs to select one or more options from a list of 5 or more items. For fewer than 5 items, consider using Radio Buttons or Checkboxes directly.

Content Guidelines

Option text should be concise and clear. The placeholder text should guide the user (e.g., 'Select a category').

Placement Rules

Typically placed within a form, associated with a

Customization Options

  • Add the `.dropdown--multi-select` class to the wrapper to enable the multi-select variant.
  • Add the `.dropdown--disabled` class and `disabled` attribute to disable the component.
  • Add the `.dropdown--open` class to programmatically open the dropdown (requires JS to manage).

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, Edge.

Device Testing

Test on mobile to confirm responsive behavior. If implementing the native fallback, test the detection logic.

Accessibility Testing

  • Keyboard navigation testing (Tab, Enter, Space, Arrow Keys) once JS is added.
  • Screen reader testing with NVDA/VoiceOver to verify roles and states are announced correctly.
  • Color contrast validation of all states.

Performance Testing

N/A for static component. For dynamic version, measure performance with large lists (>100 items).
