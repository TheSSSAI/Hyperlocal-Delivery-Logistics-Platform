Description

A set of linked input fields designed for entering a One-Time Password (OTP). It supports different lengths, provides clear states for focus, filled, and error conditions, and includes JavaScript for auto-focusing between fields and handling pasted codes.

Design System Alignment

This component utilizes the design system's tokens for colors (primary for focus, error for validation), spacing (gaps between inputs), borders (radius, width), and typography (input font size and weight).

Responsive Specifications

Mobile Adaptations

  • Input segments are sized with large touch targets (48x56px).
  • Uses `inputmode='numeric'` to bring up the numeric keyboard on mobile devices for a better user experience.

Tablet Adaptations

  • Component maintains its mobile sizing, which is already touch-friendly.

Desktop Enhancements

  • Clear focus states for keyboard navigation.
  • Supports pasting the full OTP code.

Accessibility Features

Keyboard Support

Users can Tab into the first input. JavaScript handles auto-tabbing between segments. Backspace moves focus to the previous segment.

Screen Reader Support

The component should be wrapped in a `

` with a `` (e.g., 'Enter verification code') to provide context. Each input should have `aria-label` (e.g., 'Digit 1').

Focus Management

Visible focus indicators are applied to the active segment. Focus automatically moves to the next empty segment as the user types.

Aria Attributes

  • aria-label="Digit {n}" on each input segment.
  • aria-describedby="otp-error-message" when in an error state.

Color Contrast

Input text and borders meet WCAG AA contrast requirements in all states.
