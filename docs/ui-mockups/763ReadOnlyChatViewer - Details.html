Description

A read-only component for displaying a historical log of chat messages between users for a specific order. It supports loading, empty, and populated states and is typically used in administrative or support interfaces.

Design System Alignment

This component uses the design system's color palette for message bubbles and backgrounds, spacing tokens for padding and layout, and typography for message text and timestamps. It is composed of the Spinner and MessageBubble components.

Responsive Specifications

Mobile Adaptations

  • The component is fluid and will adapt to the width of its container, making it suitable for full-screen or modal views on mobile.
  • Message bubbles have a max-width to ensure they don't span the entire screen, improving readability.

Tablet Adaptations

  • Maintains its fluid width, fitting well into multi-panel layouts common on tablets.

Desktop Enhancements

  • No specific desktop enhancements are needed; the component's design is platform-agnostic.

Accessibility Features

Keyboard Support

The main message area is scrollable via keyboard.

Screen Reader Support

The component should be contained within a landmark region with an `aria-label` (e.g., 'Chat log for order 123'). Each message bubble should have its content, sender, and timestamp read out in a logical order.

Focus Management

Since it's a read-only component, focus management is minimal. Focus is contained within the modal if it's used inside one.

Aria Attributes

  • aria-label for the main container.
  • aria-live="polite" can be used on the message container if messages are loaded dynamically after the initial view.

Color Contrast

All text colors are selected from the design system to meet WCAG 2.1 AA contrast requirements against their respective backgrounds.
