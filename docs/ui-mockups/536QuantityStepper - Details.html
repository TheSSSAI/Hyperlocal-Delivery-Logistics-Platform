Description

A form control to increment or decrement a numerical value, typically for item quantity in a cart or product listing. It is composed of two icon buttons and a read-only text input.

Design System Alignment

This component composes atomic Button (icon-only variant) and TextInput concepts. It uses design system tokens for colors (primary, neutrals), spacing, border radius, and typography to ensure visual consistency with the overall system.

Responsive Specifications

Mobile Adaptations

  • Buttons have a minimum 44x44px touch target, which is ideal for mobile.

Desktop Enhancements

Accessibility Features

Keyboard Support

Users can Tab to each button. Enter/Space activates the focused button.

Screen Reader Support

Buttons use `aria-label` to provide clear context (e.g., 'Decrease quantity', 'Increase quantity'). The input is `readonly` to indicate it is not directly editable.

Focus Management

Each button has a clear, visible focus state.

Aria Attributes

  • aria-label

Color Contrast

Icon color on the button background meets WCAG AA standards. Text in the input field has high contrast.
