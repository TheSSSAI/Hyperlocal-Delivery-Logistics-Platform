Description

A modular card component for displaying summary information about an order. It is designed with variants for new, in-progress, and historical orders, each with distinct actions and status indicators. It is a core component for vendor and admin dashboards.

Design System Alignment

The OrderCard component is built upon the atomic components and visual foundation of the design system. It utilizes tokens for colors, typography, spacing, shadows, and border-radius to ensure visual consistency with other elements like Buttons and Badges.

Responsive Specifications

Mobile Adaptations

  • The main content grid stacks into a single column for better vertical readability on narrow screens (< 768px).
  • Action buttons in the footer stack vertically and become full-width to improve touch accessibility.

Tablet Adaptations

  • The component maintains its desktop two-column layout for the content area.

Desktop Enhancements

  • Utilizes a two-column grid for a compact and scannable layout of order details.
  • Includes subtle hover effects (transform, shadow) for visual feedback.

Accessibility Features

Keyboard Support

The entire card can be a focusable element if it's a link. Internal buttons are focusable via Tab key in a logical order.

Screen Reader Support

Semantic HTML (`

`, `

`) provides structure. Detail labels and values are associated by proximity. Buttons have descriptive text.

Focus Management

Interactive elements (buttons) use the default browser focus outline. The card itself can have a focus style if it's an interactive container.

Aria Attributes

  • Not required for the base component, but buttons inside should have `aria-label` if they are icon-only.

Color Contrast

All text and badge colors meet WCAG 2.1 AA contrast requirements against their respective backgrounds.
