Description

A control for incrementing or decrementing a numerical value, typically for item quantity. It consists of a central input and two buttons for increase/decrease actions.

Design System Alignment

Combines atomic Button and TextInput components. Uses design system tokens for colors, spacing, borders, and typography to maintain consistency.

Responsive Specifications

Mobile Adaptations

  • Buttons maintain a large touch target of 44x44px for ease of use.

Tablet Adaptations

  • Component maintains its compact size and functionality.

Desktop Enhancements

  • Hover states provide clear feedback on interactive elements.

Accessibility Features

Keyboard Support

Tab to focus on each button. Enter/Space to activate.

Screen Reader Support

The main wrapper has `role="group"` and an `aria-label`. Buttons have descriptive `aria-label`s. The input is read-only and its value is announced.

Focus Management

Visible focus indicators are applied to the buttons.

Aria Attributes

  • role="group"
  • aria-label for the group
  • aria-label for each button

Color Contrast

Icon and text colors meet WCAG 2.1 AA contrast requirements.
