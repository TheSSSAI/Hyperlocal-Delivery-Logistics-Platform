Description

A composite form control for user-initiated search queries. It combines a text input with action buttons for submitting, clearing, and optionally filtering the search. It is designed to be responsive, collapsing to an icon on mobile devices.

Design System Alignment

This component utilizes the design system's tokens for form inputs, buttons, icons, spacing, and colors. It inherits the focus, error, and disabled states from the atomic TextInput and Button components.

Responsive Specifications

Mobile Adaptations

  • For the 'collapsible' variant, the component is shown as a single icon button.
  • When the icon button is tapped (simulated with :focus-within), the search bar expands to fill the available width.
  • The main input and filter buttons are hidden until the component is expanded.

Tablet Adaptations

  • The search bar is shown in its fully expanded state by default, similar to the desktop view.

Desktop Enhancements

  • The search bar is always visible and has a defined maximum width.
  • Hover and focus states are fully implemented for all interactive elements.

Accessibility Features

Keyboard Support

The entire component, including input and buttons, is keyboard navigable. Users can tab between the input and filter button. Escape key can be used to clear input.

Screen Reader Support

A visually hidden

Focus Management

The `:focus-within` pseudo-class provides a clear visual outline around the entire component when any of its interactive children have focus, improving context for keyboard users.

Aria Attributes

  • aria-label for icon buttons
  • for/id pairing for label and input

Color Contrast

All text and icons adhere to the WCAG 2.1 AA contrast ratio guidelines inherited from the design system's color tokens.
