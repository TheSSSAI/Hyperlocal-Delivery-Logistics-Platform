Description

A form control composed of two icon buttons and a text input to allow users to increment or decrement a numerical value, typically for item quantity in a cart. It handles states for minimum and maximum quantity limits.

Design System Alignment

Uses design system tokens for colors (primary, neutrals), spacing, borders, and typography. It is composed of the atomic Button, Icon, and TextInput components.

Responsive Specifications

Mobile Adaptations

  • Ensures touch targets for buttons are a minimum of 44x44px, which is suitable for mobile interactions.
  • The component's fixed size makes it predictable in various mobile layouts.

Tablet Adaptations

  • Maintains mobile sizing, which is also appropriate for tablet touch interfaces.

Desktop Enhancements

  • Includes hover and focus states that are primarily discoverable on desktop with a mouse or keyboard.

Accessibility Features

Keyboard Support

The component is keyboard navigable. Users can Tab to the minus button, then the input, then the plus button. Buttons are activated with Enter/Space.

Screen Reader Support

Buttons use descriptive `aria-label` attributes (e.g., 'Decrease quantity', 'Increase quantity'). The input is readonly but its value is announced.

Focus Management

A `focus-within` style on the parent container provides a clear visual group focus, meeting WCAG requirements.

Aria Attributes

  • aria-label on buttons provides context for icon-only controls.
  • aria-labelledby can be used on the parent `div` to link it to an external label like the product name.

Color Contrast

Text and icon colors meet WCAG AA contrast ratios against their backgrounds in all states.
