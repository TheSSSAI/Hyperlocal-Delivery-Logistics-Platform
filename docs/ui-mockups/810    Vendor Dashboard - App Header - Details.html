Responsive Specifications

Desktop Layout

  • Fixed-position header at the top of the viewport.
  • Content is aligned using flexbox with 'space-between'.
  • Hamburger menu icon is hidden.

Tablet Adaptations

  • Maintain desktop layout but with potentially increased touch target sizes for icons.
  • Hamburger menu becomes visible at the tablet breakpoint.

Mobile Adaptations

  • Hamburger menu icon is visible and is the primary navigation trigger.
  • Spacing and icon sizes are optimized for smaller viewports.

Accessibility Features

Keyboard Navigation

Complete and logical tab order through all interactive elements (hamburger menu, notification bell, profile avatar).

Screen Reader Support

Semantic

and
landmarks are used. All icon-only buttons have descriptive aria-labels.

Focus Management

Visible focus states are applied to all interactive elements. When the profile dropdown is open, focus should be manageable within it.

Aria Attributes

  • aria-label for icon buttons
  • aria-haspopup='true' for the avatar
  • aria-expanded for the avatar
  • role='menu' and role='menuitem' for the dropdown

Color Contrast

All text and icons meet WCAG 2.1 AA contrast requirements against their backgrounds.
