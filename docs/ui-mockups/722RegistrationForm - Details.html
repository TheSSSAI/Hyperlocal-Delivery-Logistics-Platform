Description

A multi-step, multi-variant form organism used for onboarding new Customers, Vendors, and Riders. It composes several molecular components like FormGroup, OTPInput, and FileUpload to gather required information.

Design System Alignment

This organism is a composition of molecular and atomic components from the design system. It uses FormGroup for layout, Button for actions, OTPInput for verification, and FileUpload for documents. It strictly adheres to the design system's tokens for typography, color, spacing, and form styling to ensure a cohesive user experience.

Responsive Specifications

Mobile Adaptations

  • The form adopts a single-column layout which is standard for mobile.
  • The component will stretch to the full width of the viewport on mobile screens to maximize readability and ease of input.

Tablet Adaptations

  • The form maintains a single-column layout but is constrained by a max-width for better readability.

Desktop Enhancements

  • The form is centered on the page with a max-width to prevent lines of text from becoming too long, which improves readability.

Accessibility Features

Keyboard Support

The form is fully navigable using the Tab key. All interactive elements (inputs, buttons, file uploads) are focusable and operable.

Screen Reader Support

The form is wrapped in a `

`, ``) are used to group related sections (e.g., 'Business Details', 'Documents'). All inputs have associated labels.

Focus Management

Focus is managed by the browser's default behavior. JavaScript is used within the OTPInput component to manage focus between digit fields.

Aria Attributes

  • aria-describedby is used to link error messages to inputs within the FormGroup component.
  • aria-live='polite' is used for the success message to announce it to screen readers.

Color Contrast

All text, borders, and UI elements meet WCAG 2.1 AA contrast requirements.
