Description

A modal dialog is an overlay that requires user interaction. It's used for confirmations, forms, or displaying critical information without losing the context of the underlying page. It includes variants for simple confirmations, forms, and informational messages.

Design System Alignment

This organism component combines atomic elements like Button and Icon, and molecular components like FormGroup. It uses the design system's tokens for color, spacing, typography, shadows, and responsive breakpoints to ensure a consistent overlay experience.

Responsive Specifications

Mobile Adaptations

  • On screens < 768px, the modal becomes full-screen, taking up 100% of the viewport width and height.
  • Border radius is removed on mobile for an edge-to-edge experience.
  • Footer action buttons stack vertically and become full-width for better touch accessibility.

Tablet Adaptations

  • The modal maintains its desktop appearance with a max-width and centered alignment.

Desktop Enhancements

  • The modal appears as a floating dialog over a semi-transparent backdrop, maintaining the context of the page behind it.

Accessibility Features

Keyboard Support

The modal should be fully keyboard navigable. Focus should be trapped within the modal when open. The Escape key should close the modal.

Screen Reader Support

The main container has `role="dialog"` and `aria-modal="true"`. The title is associated with the dialog via `aria-labelledby`.

Focus Management

When the modal opens, focus should be programmatically moved to the first interactive element inside it (e.g., the close button or a form field).

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby="modal-title-id"
  • aria-describedby="modal-description-id" (for info modals)

Color Contrast

All text and interactive elements within the modal meet WCAG 2.1 AA contrast requirements.
