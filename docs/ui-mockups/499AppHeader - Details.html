Description

The main application header, providing navigation, branding, and key user actions. It has distinct variants for each of the four user-facing applications: Customer App, Rider App, Vendor Dashboard, and Admin Dashboard.

Design System Alignment

This organism component composes multiple molecular and atomic components like Button, Icon, Link, SearchBar, and Avatar. It utilizes the design system's tokens for color, spacing, typography, and shadows to establish a consistent brand and navigation experience across all platforms.

Responsive Specifications

Mobile Adaptations

  • For web dashboards (Vendor, Admin), the main navigation links are hidden, and a hamburger menu icon is shown.
  • Clicking the hamburger icon reveals a full-width dropdown menu with navigation links.
  • Mobile app headers (Customer, Rider) are designed for mobile-first and do not change significantly across screen sizes.

Tablet Adaptations

  • Web dashboard headers may still show the hamburger menu in portrait mode but switch to the desktop view in landscape mode, depending on the breakpoint (1024px).

Desktop Enhancements

  • Web dashboard headers display the logo, a full horizontal navigation link set, and user actions (search, profile).
  • The hamburger menu is hidden on screens wider than 1024px.

Accessibility Features

Keyboard Support

All interactive elements (logo, links, buttons, avatar) are focusable and activatable via keyboard.

Screen Reader Support

Uses semantic `

` and `
` elements. The mobile menu toggle button has `aria-controls` and `aria-expanded` attributes. Icon buttons have descriptive `aria-label`s.

Focus Management

Visible focus indicators are applied to all interactive elements. When the mobile menu is open, focus should be trapped within it.

Aria Attributes

  • role="banner"
  • aria-label for main navigation
  • aria-controls
  • aria-expanded

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast requirements against their respective backgrounds.
