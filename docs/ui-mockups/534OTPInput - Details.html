Description

A set of segmented input fields for entering a One-Time Password (OTP). It provides automatic focus shifting between inputs, backspace handling, and paste support for a seamless user experience. It comes in 4-digit and 6-digit variants and supports default, focus, filled, and error states.

Design System Alignment

This component utilizes the design system's tokens for form inputs, including colors for borders (default, focus, error), spacing between segments, border radius, and typography for the digits. It inherits the base styling of the atomic TextInput component.

Responsive Specifications

Mobile Adaptations

  • Input segments are touch-friendly with a minimum height of 52px.
  • Gap between segments is reduced to save horizontal space.
  • Uses `inputmode='numeric'` and `pattern='[0-9]*'` to trigger the numeric keyboard on mobile devices for a better UX.

Tablet Adaptations

  • Maintains desktop sizing and spacing.

Desktop Enhancements

  • Clear focus states with box-shadow for keyboard navigators.
  • Supports pasting the entire OTP code from the clipboard.

Accessibility Features

Keyboard Support

Users can tab into the first input. Entering a digit automatically moves focus to the next input. Backspace on an empty input moves focus to the previous one.

Screen Reader Support

Aria-label on the container (e.g., 'One-Time Password') and individual inputs (e.g., 'OTP digit 1') provides context. A hidden master input holds the full value for form submission.

Focus Management

Focus is programmatically managed via JavaScript to create a seamless entry experience without requiring the user to manually switch between inputs.

Aria Attributes

  • aria-label for the container and each input.
  • aria-invalid='true' can be applied to inputs in the error state.

Color Contrast

The digit color against the input background and the border colors meet WCAG AA contrast requirements.
