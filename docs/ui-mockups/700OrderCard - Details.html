Description

A self-contained card component that displays summary information for a single order. It adapts its content and actions based on the order's state, such as a new incoming order, an order in progress, or a historical order.

Design System Alignment

The OrderCard component is built using atomic design system elements. It utilizes the primary color palette for actions and semantic colors for status badges. Spacing follows the 8px grid system, and typography is derived from the established font scale. The card itself uses the default card styles for background, shadow, and border-radius.

Implementation Notes

Embedded Css Architecture

All styles are embedded, leveraging CSS custom properties for design tokens. BEM is used for class naming (`.order-card`, `.order-card__header`).

Component Organization

The HTML is structured semantically with `

`, `
`, and `
`. A `data-variant` attribute is used to switch between the 'new-order', 'in-progress', and 'history' styles.

Responsive Design

A mobile-first approach is used. The base styles are for mobile, and a `max-width` media query is used to adapt the layout for smaller screens by stacking elements vertically.

Accessibility Compliance

WCAG 2.1 AA is addressed through semantic tags, appropriate ARIA attributes for labeling, and clear focus states.

Browser Support

Uses modern CSS like Flexbox and custom properties, compatible with all evergreen browsers.

Usage Guidelines

When To Use

Use to display a summary of an order in a list view, such as a vendor's dashboard, an admin's order management screen, or a customer's order history.

Content Guidelines

Keep details concise. The card should be a summary that links to a more detailed view. Use the appropriate variant based on the order's status.

Placement Rules

Typically placed in a single column list on mobile and a multi-column grid or single column list on desktop.

Customization Options

  • Use `data-variant="new-order"` for incoming orders requiring action.
  • Use `data-variant="in-progress"` for active orders.
  • Use `data-variant="history"` for completed or cancelled orders.
