Description

A form control for selecting a rating from 1 to 5 stars, or for displaying a read-only rating value. It is designed to be accessible and touch-friendly.

Design System Alignment

Uses design system tokens for colors (warning, gray), spacing, and focus states. The star icon is a dependency from the Icon component set.

Responsive Specifications

Mobile Adaptations

  • Increased font size for icons to create larger touch targets (min 44x44px).
  • Added padding around interactive stars to increase tappable area.

Tablet Adaptations

  • Maintain desktop sizing with sufficient touch targets.

Desktop Enhancements

  • Includes precise hover effects for mouse interaction.

Accessibility Features

Keyboard Support

For the interactive variant, users can use Tab to navigate to the star group and arrow keys to select a rating, as it's built on native radio buttons.

Screen Reader Support

The interactive version uses a `fieldset` and `legend` to group the controls. Each star is a `label` for a visually hidden radio `input` with a value (e.g., '1 star'). The read-only version uses `aria-label` to announce the rating (e.g., 'Rated 3.5 out of 5 stars').

Focus Management

A clear, visible focus outline is applied to the star `label` when its corresponding radio button is focused.

Aria Attributes

  • aria-label for read-only variant
  • aria-hidden for decorative-only stars in read-only variant
  • role="group" on the fieldset (implicit)

Color Contrast

The warning color for selected stars has sufficient contrast against a white background. The gray for unselected stars is a non-critical indicator.
