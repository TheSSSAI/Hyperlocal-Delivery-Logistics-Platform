Description

A versatile search input component for finding stores or items. It combines a TextInput, Button, and Icon, and supports states for focus, loading, and text input. It can be paired with a filter button for advanced search.

Design System Alignment

Utilizes the design system's color palette, typography, spacing, and form input specifications to ensure a consistent look and feel with other platform components.

Responsive Specifications

Mobile Adaptations

  • On focus, the search bar expands to cover the full width of its container or screen, overlaying other content to maximize input space.
  • Touch targets for all buttons (icons, filters) are maintained at a minimum of 44px.

Tablet Adaptations

  • Maintains desktop-like appearance and functionality.

Desktop Enhancements

  • Standard inline search bar behavior.
  • Hover states for filter and icon buttons.

Accessibility Features

Keyboard Support

The form can be submitted with the Enter key. All buttons are focusable via Tab key and activatable with Enter/Space.

Screen Reader Support

The main wrapper has `role="search"`. The input is associated with a hidden `

Focus Management

A highly visible focus ring is applied to the entire component wrapper (`:focus-within`) and individual buttons (`:focus-visible`).

Aria Attributes

  • role="search" on the form
  • aria-label on icon buttons

Color Contrast

Placeholder text and icon colors meet WCAG AA contrast ratios against the background.
