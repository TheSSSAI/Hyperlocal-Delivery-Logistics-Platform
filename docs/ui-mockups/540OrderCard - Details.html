Description

A self-contained card component that displays a summary of an order. It has variants for different stages of the order lifecycle, such as new, in-progress, and historical orders. It aggregates atomic components like Badges and Buttons to present information and actions.

Design System Alignment

This component heavily uses the design system's tokens for color, spacing, typography, and borders. It's built upon the atomic Button and Badge components, ensuring visual consistency with the rest of the platform.

Responsive Specifications

Mobile Adaptations

  • Header and footer elements stack vertically to save horizontal space.
  • Action buttons in the footer expand to full width for better touch targets.
  • The main body grid reduces to two columns for better readability on smaller screens.

Tablet Adaptations

  • The desktop layout is maintained, but with sufficient spacing for touch interaction.

Desktop Enhancements

  • A multi-column grid layout for the body allows for a quick scan of all key order details.
  • A subtle hover effect (lift and shadow) provides interactive feedback.

Accessibility Features

Keyboard Support

All interactive elements (buttons) are keyboard-focusable and can be activated with Enter/Space.
