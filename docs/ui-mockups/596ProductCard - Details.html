Description

Displays a product's key information, including image, name, price, stock status, and actions. It supports both grid and list layouts and adapts responsively.

Design System Alignment

Utilizes the design system's color palette, typography, spacing, and border-radius tokens. It is composed of atomic components like Button, Badge, and QuantityStepper.

Responsive Specifications

Mobile Adaptations

  • Defaults to a list-view variant for better vertical scrolling.
  • Image is on the left, content on the right.
  • Full width of the container.

Tablet Adaptations

  • Transitions to a grid-view layout, showing multiple cards per row.
  • Image is on top, content below.

Desktop Enhancements

  • Uses grid-view with more columns.
  • Hover effects provide visual feedback.

Accessibility Features

Keyboard Support

The main card link is focusable. Buttons and quantity steppers within the card are also focusable and operable with Enter/Space.

Screen Reader Support

Uses semantic elements like `

` and `

`. Disabled state is conveyed via text ('Out of Stock' badge) and disabled buttons.

Focus Management

A clear, visible focus ring is applied to the main card link and interactive elements within.

Aria Attributes

  • `aria-disabled='true'` is applied to buttons in the disabled state.

Color Contrast

Text and background colors meet WCAG 2.1 AA requirements. Status is communicated with both color and text via the Badge component.
