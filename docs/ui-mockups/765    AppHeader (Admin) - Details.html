Responsive Specifications

Desktop Layout

  • Full-width header with logo on the far left.
  • Search bar positioned in the center or left-aligned next to the logo.
  • Utility navigation (notifications, user profile) on the far right.

Tablet Adaptations

  • Layout remains similar to desktop, but spacing may be slightly reduced.
  • Search bar may collapse into an icon on smaller tablet sizes.

Mobile Adaptations

  • A hamburger menu icon is displayed on the left to toggle the main navigation.
  • The logo is centered or placed next to the hamburger icon.
  • The search bar is typically hidden and accessed via a dedicated search icon.

Accessibility Features

Keyboard Navigation

Complete and logical tab order through all interactive elements: hamburger menu, logo, search, notification bell, user avatar.

Screen Reader Support

Semantic

Focus Management

Visible focus states are applied to all interactive elements. When a dropdown or mobile menu is opened, focus is trapped within it.

Aria Attributes

  • aria-label
  • aria-controls
  • aria-expanded
  • role='button'
  • role='navigation'

Color Contrast

All text, icons, and interactive elements meet WCAG 2.1 AA contrast requirements against the header background.
