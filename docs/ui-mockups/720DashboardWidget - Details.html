Description

A self-contained card for displaying a specific piece of data or visualization on a dashboard. It supports multiple variants like Key Performance Indicator (KPI) cards, line charts, and simple lists, and handles its own loading and error states.

Design System Alignment

This organism component uses the design system's foundational elements for its structure and styling. It utilizes the default card specification for its container, typography tokens for all text, and spacing tokens for internal layout. It is composed of atomic components like 'Spinner' and 'Alert' for its feedback states, ensuring visual consistency across the platform.

Responsive Specifications

Mobile Adaptations

  • Widgets stack vertically in a single column, managed by the parent grid container.
  • Internal padding and typography are slightly reduced to be more space-efficient.

Tablet Adaptations

  • Widgets are typically displayed in a two or three-column grid, managed by the parent container.

Desktop Enhancements

  • Widgets are displayed in a multi-column grid, providing a dense, at-a-glance overview of platform metrics.

Accessibility Features

Keyboard Support

Interactive elements within a widget, such as a settings dropdown, are keyboard-focusable and operable.

Screen Reader Support

Each widget is contained within a landmark element (e.g., `

`) with an `aria-labelledby` attribute pointing to its title. The loading state is announced via `aria-busy='true'`, and the error state uses `role='alert'`. Charts include a visually hidden data table for screen reader access.

Focus Management

Focus is managed within the widget if it contains interactive elements.

Aria Attributes

  • aria-labelledby
  • aria-busy="true"
  • role="alert"

Color Contrast

All text and data visualizations meet WCAG 2.1 AA contrast requirements.
