Description

A versatile overlay component for displaying contextual information, forms, or critical confirmations. It interrupts the user's workflow to focus their attention on a specific task or piece of information.

Design System Alignment

This component is an organism composed of atomic components like Button and Icon, and molecular components like FormGroup. It heavily relies on the design system's tokens for color, spacing, typography, border-radius, and shadows to maintain visual consistency.

Responsive Specifications

Mobile Adaptations

  • The modal appears as a bottom sheet, animating up from the bottom of the screen.
  • It takes up the full width of the screen.
  • The maximum height is constrained to 80-90% of the viewport height to allow for dismissal by tapping the backdrop.
  • The border-radius is adjusted to be sharp at the bottom corners.

Tablet Adaptations

  • Maintains the desktop-style centered dialog, as screen real estate is sufficient.

Desktop Enhancements

  • Appears as a centered dialog with a backdrop overlay.
  • Includes a subtle scale and fade-in animation for a smooth entrance.

Accessibility Features

Keyboard Support

The modal must trap focus. All interactive elements (close button, form inputs, action buttons) must be reachable via Tab key. The Escape key must close the modal.

Screen Reader Support

The main dialog element uses `role='dialog'` or `role='alertdialog'` for critical confirmations. `aria-modal='true'` is essential. `aria-labelledby` points to the modal title, and `aria-describedby` can point to the main content body.

Focus Management

When opened, focus must be programmatically moved to the first focusable element inside the modal (e.g., the close button or a form field). When closed, focus must be returned to the element that triggered the modal.

Aria Attributes

  • role='dialog'
  • role='alertdialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet WCAG 2.1 AA contrast requirements. Icons for variants (e.g., danger) use semantic colors from the design system.
