Description

A molecular component that groups a label, a form input, and an optional error message. It manages layout, spacing, and accessibility linking between these elements.

Implementation Notes

Embedded Css Architecture

All styles, including dependencies like Label and TextInput, are embedded in the HTML file. Design tokens are defined in `:root`.

Component Organization

The FormGroup acts as a wrapper. The state is controlled by adding a modifier class (`.form-group--error`) to the wrapper, which then styles its children.

Responsive Design

The component is inherently responsive due to its simple, vertical flow. It will adapt to the width of its parent container.

Accessibility Compliance

Accessibility is achieved through correct HTML attributes (`for`, `id`, `aria-describedby`) which link the label and error message to the input.

Browser Support

Uses modern CSS properties that are compatible with all target browsers.

Organism Map

The FormGroup is a fundamental building block for various forms across the platform, including the RegistrationForm (for all user types), the Vendor Product Management forms (add/edit product), and various settings forms in the Admin Dashboard like the Geofencing tool.

Usage Guidelines

When To Use

Use this component whenever a form input needs a label and may need to display a validation error message. It should be used for all standard form fields.

Content Guidelines

The label text should be concise and clear. The error message should be actionable and explain how to fix the error.

Placement Rules

Typically placed vertically within a `

` element or a more complex form organism.

Customization Options

  • Add `.label--required` class to the label to show the required indicator.
  • Add `.form-group--error` to the main wrapper to activate the error state.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, Edge.

Device Testing

Verify layout on mobile, tablet, and desktop viewports.

Accessibility Testing

  • Verify that clicking the label focuses the input.
  • Use a screen reader (NVDA, VoiceOver) to confirm that the label and error message are read out when the input is focused.
  • Check color contrast for error text.
