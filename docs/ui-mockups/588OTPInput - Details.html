Description

A set of input fields designed for entering a One-Time Password (OTP). It provides a user-friendly interface for entering sequential digits and handles focus management, pasting, and validation states automatically.

Design System Alignment

Utilizes the design system's tokens for form inputs, including border styles, colors for default, focus, and error states, spacing, and typography. It builds upon the atomic `TextInput` component's visual language.

Responsive Specifications

Mobile Adaptations

  • Input segments are sized to be easily tappable (touch-friendly).
  • On very small screens (<480px), the gap and size of the inputs are slightly reduced to ensure the component fits without wrapping.

Tablet Adaptations

  • Maintains the default sizing, which is already touch-friendly.

Desktop Enhancements

  • Supports rapid entry via keyboard, including pasting the full code.

Accessibility Features

Keyboard Support

Users can type digits sequentially. Backspace moves focus to the previous input. Users can paste the entire code at once.

Screen Reader Support

The component can be wrapped in a `

` with a `` to provide context. Each input has an `aria-label` like 'OTP Digit 1'. The `aria-describedby` attribute links the component to instruction and error messages.

Focus Management

Focus automatically moves to the next input field as the user types, and to the previous field on backspace, managed by embedded JavaScript.

Aria Attributes

  • aria-label="OTP Digit X"
  • aria-describedby="otp-instructions otp-error-message"
  • autocomplete="one-time-code"

Color Contrast

Text and border colors meet WCAG 2.1 AA requirements for contrast.

Implementation Notes

Embedded Css Architecture

Styles are embedded in the `` using CSS custom properties for tokens. A BEM naming convention (`.otp-input`, `.otp-input__field`, `.is-error`) is used for clarity and scoping.

Component Organization

The component consists of a container div and a series of `` elements. A small, embedded JavaScript is required for the interactive behavior (focus shifting, pasting) which is essential for a good user experience.

Responsive Design

The component is inherently responsive due to flexbox layout and fixed-size inputs. A media query is included to slightly scale down the component on very narrow mobile screens.

Accessibility Compliance

WCAG 2.1 AA compliance is achieved through semantic HTML structure, proper ARIA attributes, and JavaScript-managed focus behavior. The `autocomplete` attribute helps password managers auto-fill OTPs.

Browser Support

Works on all modern evergreen browsers. The CSS is widely supported.

Usage Guidelines

When To Use

Use for any user flow requiring a One-Time Password, such as login, registration, or confirming sensitive actions like changing bank details.

Content Guidelines

The number of inputs should match the length of the OTP being sent (typically 4 or 6 digits).

Placement Rules

Should be placed centrally within a form, accompanied by a clear label, instructions, and an error message container.

Customization Options

  • The number of input fields can be configured by changing the `data-digits` attribute on the container.
  • Add the `.is-error` class to the main `.otp-input` container to apply the error styling to all fields.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, and Edge.

Device Testing

Test on both iOS and Android mobile devices to ensure the numeric keyboard appears correctly and touch interactions are smooth.

Accessibility Testing

  • Verify keyboard navigation (typing, backspace).
  • Test pasting a full OTP code from the clipboard.
  • Use screen readers (VoiceOver, TalkBack) to ensure the labels, instructions, and error messages are announced correctly.
