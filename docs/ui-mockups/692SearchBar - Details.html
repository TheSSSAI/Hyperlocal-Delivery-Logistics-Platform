Description

A molecular component combining a text input, icon, and optional buttons for search functionality. It supports various states including focus, loading, and can be configured with or without a filter button. It is designed to be responsive, collapsing to an icon on mobile devices.

Design System Alignment

This component is built by composing the atomic 'TextInput', 'Button', and 'Icon' components. It uses the design system's color tokens for interactivity and states (primary for focus), spacing tokens for layout, and border-radius for shape. Typography for placeholder and input text is also derived from the design system.

Responsive Specifications

Mobile Adaptations

  • The search bar collapses into a circular icon-only button to save space in the header.
  • On tap/focus, the component expands to a full-width input field, overlaying other header elements if necessary.
  • Filter and clear buttons are hidden in the collapsed state and appear upon expansion.

Tablet Adaptations

  • May retain the collapsed icon state on portrait tablets or expand to the full bar on landscape, depending on available header space.

Desktop Enhancements

  • The full search bar with input, icon, and optional filter button is always visible.
  • Supports hover states for interactive elements.
  • Can display a dropdown of search results below the input.

Accessibility Features

Keyboard Support

The component and its internal buttons are focusable via the Tab key. The input accepts text entry. Results list can be navigated with arrow keys (with JS).

Screen Reader Support

The input has an associated implicit or explicit label. Icon-only buttons have `aria-label`. The search icon is decorative (`aria-hidden='true'`). The results dropdown has appropriate ARIA roles.

Focus Management

Focus is managed using `:focus-within` to style the entire container when the input is active. On mobile, focus on the icon button expands the input.

Aria Attributes

  • aria-label for icon buttons
  • aria-busy on the container during the loading state
  • role='listbox', role='option' for the results dropdown (with JS)

Color Contrast

All text and borders meet WCAG 2.1 AA contrast requirements.
