Description

A multi-step, responsive form for user registration. It provides distinct variants for different user types (Customer, Vendor, Rider), each with specific fields and document upload requirements. The component manages default, submitting, and success states.

Design System Alignment

This organism is composed of molecular components like FormGroup, FileUpload, and OTPInput, and atomic components like Button and Label. It adheres strictly to the design system's tokens for typography, color, spacing, and form control styling.

Responsive Specifications

Mobile Adaptations

  • The form maintains a single-column layout, which is inherently mobile-friendly.
  • Padding and font sizes are adjusted slightly for smaller screens to optimize space.

Tablet Adaptations

  • The form maintains its single-column layout but within a wider, centered container.

Desktop Enhancements

  • On larger screens, the form has a maximum width to ensure readability and is centered on the page.
