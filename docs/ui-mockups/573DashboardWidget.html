Screen Variations Documentation

Variant Name: "KPI Card"

Used for displaying a single, important metric. It emphasizes a large numerical value and can optionally show a trend comparison. It's ideal for at-a-glance monitoring of key business health indicators like sales, user counts, or conversion rates.

Variant Name: "Line Chart"

Used for visualizing time-series data to show trends over a period. This example uses a simplified bar-like representation to demonstrate layout, but in a real application, it would be replaced by a proper charting library like Chart.js or Recharts to render lines or bars.

Variant Name: "List Widget"

Used for displaying a short, ranked list of items, such as top-selling products, top-performing vendors, or recent activities. It provides a quick summary without the need for a full data table.

State Name: "With Data"

The default, successful state of the widget after data has been fetched and rendered correctly. All content is visible and accurately displayed.

State Name: "Loading State"

Indicates that data for the widget is currently being fetched from the server. A skeleton loader is used to provide a low-contrast placeholder that mimics the final layout, reducing perceived load time and preventing content layout shift.

State Name: "Error State"

Indicates that the data for the widget could not be fetched. It uses the `Alert` component to display a clear, user-friendly error message within the widget's content area, allowing the rest of the dashboard to function normally.
