Description

A component for handling file selection and uploads. It supports a simple file display variant and an image preview variant, with distinct visual states for idle, uploading, success, and error conditions. It is designed to be full-width on mobile devices.

Design System Alignment

This component utilizes the design system's tokens for colors (primary, semantic error, neutrals), spacing, border-radius, and typography. It also incorporates atomic components like Button, Spinner, and Alert for its various states and actions.

Responsive Specifications

Mobile Adaptations

  • The component container becomes full-width to maximize screen real estate.

Tablet Adaptations

  • Maintains desktop sizing but with larger touch targets for interactive elements like the remove button.

Desktop Enhancements

  • Includes hover effects on the dropzone to provide better affordance.
  • Fixed max-width for better integration into multi-column layouts.

Accessibility Features

Keyboard Support

The hidden file input is focusable via the Tab key, triggering the browser's native file picker. The 'Remove' button is also focusable and can be activated with Enter/Space.

Screen Reader Support

The component is wrapped in a `

Focus Management

A clear, visible focus ring is applied to the dropzone when the hidden input receives focus, meeting WCAG requirements.

Aria Attributes

  • The `Alert` component in the error state has `role='alert'`.
  • The `Spinner` component in the uploading state is accompanied by text and could use `aria-busy='true'` on the container.

Color Contrast

All text and interactive elements adhere to the WCAG 2.1 AA contrast ratio of at least 4.5:1.
