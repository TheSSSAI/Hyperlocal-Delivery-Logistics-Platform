Description

A self-contained organism for real-time text communication between users (Customer, Vendor, Rider) within the context of an active order. It includes a message history, an input area with quick replies, and supports various states like active, read-only, loading, and error.

Design System Alignment

This component orchestrates several atomic and molecular components, including TextInput, Button, Avatar, and Badge. It heavily relies on the design system's tokens for typography, colors, spacing, and border-radius to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The component is designed to fill the screen height on mobile.
  • The layout adapts to the on-screen keyboard by using a flexbox container, allowing the message body to shrink while the input footer stays visible.

Tablet Adaptations

  • The component can be used in a split-view layout, for example, next to a list of orders in the Admin/Vendor dashboard.
  • No specific style changes are needed; the flexbox layout is inherently adaptable.

Desktop Enhancements

  • Hover states on quick-reply buttons provide better feedback for mouse users.

Accessibility Features

Keyboard Support

The chat input area is fully keyboard navigable. Users can Tab from the textarea to the Send button. Quick replies are also focusable.

Screen Reader Support

The message history container has `role="log"` and `aria-live="polite"` so that new messages are announced. The input textarea is associated with a visually hidden label. All buttons have descriptive text or `aria-label` attributes.

Focus Management

Focus is managed within the component. When the chat opens, focus should be set to the text input area.

Aria Attributes

  • role="log"
  • aria-live="polite"
  • aria-label for buttons

Color Contrast

All text and background color combinations meet WCAG 2.1 AA contrast requirements.
