Description

A form control component that allows users to increment or decrement a numerical value. It is composed of two buttons and a text input to display the current quantity. The component has states to handle minimum and maximum value limits.

Design System Alignment

This component is built by composing the atomic 'Button', 'Icon', and 'TextInput' components. It uses the design system's color tokens for interactivity and states, spacing tokens for layout, and border-radius for shape. It specifically adheres to the requirement for large touch targets on mobile.

Responsive Specifications

Mobile Adaptations

  • Buttons must have a minimum width and height of 44px to meet large touch target requirements.
  • The input text size is maintained or slightly increased for readability.

Tablet Adaptations

  • Component size remains consistent with mobile and desktop.

Desktop Enhancements

  • Includes hover states for the increment and decrement buttons.

Accessibility Features

Keyboard Support

The component is fully keyboard navigable. Users can Tab to each button and the input. Enter/Space activates the buttons.

Screen Reader Support

Buttons have descriptive `aria-label`s (e.g., 'Increase quantity', 'Decrease quantity'). The input is readonly but can be associated with a label for context.

Focus Management

Each interactive element (`+` and `-` buttons) has a clear, visible focus state.

Aria Attributes

  • aria-label on buttons.
  • disabled attribute is used to programmatically disable buttons at min/max limits.

Color Contrast

Icon and text colors meet WCAG 2.1 AA contrast requirements against their backgrounds.
