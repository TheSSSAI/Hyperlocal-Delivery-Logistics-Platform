Description

A component for handling file uploads. It provides a drop zone or file selection interface, and displays the state of the upload (idle, uploading, success, error). It has variants for general file uploads and specialized image uploads with a preview.

Design System Alignment

This component composes atomic components like 'Button', 'Spinner', and 'Alert'. It utilizes the design system's tokens for colors (primary, semantic), spacing, typography, borders, and shadows to maintain visual consistency with other form elements.

Responsive Specifications

Mobile Adaptations

  • The component becomes full-width on mobile to maximize the touch area and use available screen space.
  • Padding is slightly reduced to be more space-efficient on smaller screens.

Tablet Adaptations

  • Maintains the desktop layout, which is already sufficiently large for touch interactions.

Desktop Enhancements

  • Includes hover states to provide clear visual feedback on interactivity.
  • Supports drag-and-drop functionality (requires JavaScript implementation).

Accessibility Features

Keyboard Support

The hidden file input is keyboard-focusable. Users can press Enter or Space to open the native file selector.

Focus Management

A clear focus-visible state is applied to the main container when the hidden input is focused.

Aria Attributes

  • `aria-describedby` can be used to link the component to instructional text.
  • An associated ARIA live region (`aria-live='polite'`) should be used to announce status changes.

Color Contrast

All text and icons meet WCAG 2.1 AA contrast requirements against their respective backgrounds.
