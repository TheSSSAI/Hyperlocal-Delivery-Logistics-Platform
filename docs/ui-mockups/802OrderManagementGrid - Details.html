Description

A responsive grid layout for managing orders across different stages (New, Preparing, Ready). It's the central component of the vendor dashboard, designed for at-a-glance operational awareness and quick actions. It is composed of molecular OrderCard components and atomic Spinners for its states.

Design System Alignment

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 1024px, the three columns stack vertically into a single column.
  • The height of each column's card list becomes dynamic to accommodate vertical scrolling on the page.

Tablet Adaptations

  • Behaves the same as mobile (stacked vertically) to ensure readability and avoid horizontal scrolling of the entire grid.

Desktop Enhancements

  • Displays a three-column layout for a comprehensive overview of the order pipeline.
  • Utilizes fixed height for card lists to allow independent scrolling within each column.

Accessibility Features

Keyboard Support

The grid and its columns are navigable. Interactive elements within the contained OrderCards (buttons) are focusable and operable via keyboard.

Screen Reader Support

Column headers (h3) provide semantic structure. The loading state uses `aria-busy='true'` to inform screen readers. The empty state uses standard text that is announced.

Focus Management

Focus is managed by the browser's default document flow. Focus states for buttons are provided by the atomic Button component.

Aria Attributes

  • aria-busy='true' on the grid during the loading state.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast ratio requirements, ensuring readability.
