Description

Displays a single product with an image, name, price, stock status, and an action button. It is designed to be used in product listings and search results, adapting its layout for different screen sizes.

Design System Alignment

This component combines several atomic components (Badge, Button) and uses foundational design system tokens for color, typography, spacing, and shadows to ensure visual consistency.

Responsive Specifications

Mobile Adaptations

  • Cards use a vertical layout (image on top, content below) to fit narrow screens.
  • The 'grid-view' on desktop becomes a 'list-view' on mobile implicitly through this layout change.

Tablet Adaptations

  • Can either maintain the mobile (vertical) layout or switch to the desktop (horizontal) layout depending on the grid context.

Desktop Enhancements

  • Cards use a horizontal layout (image on left, content on right) for better space utilization in multi-column grids.
  • Hover effects provide visual feedback.

Accessibility Features

Keyboard Support

The entire card is focusable using the Tab key if it's a link. Interactive elements within (like the button) are also focusable.

Screen Reader Support

Semantic `article` tag identifies the card as a self-contained unit. The product title is an `h3` for proper heading structure. Images have `alt` text.

Focus Management

Visible focus indicators are applied using a box-shadow to ensure clarity.

Aria Attributes

  • `aria-label` on action buttons to provide context, e.g., 'Add Spicy Chicken Ramen to cart'.

Color Contrast

All text meets WCAG 2.1 AA contrast requirements. Status badges use text in addition to color to convey meaning.
