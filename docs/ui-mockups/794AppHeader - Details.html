Description

A responsive, fixed application header for web dashboards. It includes a logo, a placeholder for navigation or title, and utility controls for notifications and user profile management. It adapts for mobile viewports by integrating with a sidebar navigation trigger.

Design System Alignment

This component uses the design system's color palette for background and text, spacing tokens for padding and height, and typography for text elements. It is composed of atomic and molecular components like Button, Icon, Avatar, and Dropdown.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 768px, a hamburger menu icon is displayed to toggle the main sidebar navigation.
  • The user's name and dropdown chevron are hidden, showing only the avatar for a more compact profile button.

Tablet Adaptations

  • The hamburger menu icon is hidden, and the full desktop layout is used.

Desktop Enhancements

  • Displays the full layout with logo, utility navigation (notifications, profile), and the user's name next to their avatar.
  • Utilizes hover effects for interactive elements like the profile dropdown.

Accessibility Features

Keyboard Support

All interactive elements (menu toggle, notification bell, profile dropdown) are focusable and can be activated using Enter/Space keys.

Screen Reader Support

The hamburger menu toggle has an `aria-label` of 'Open navigation menu'. The notification bell button has an `aria-label` describing its state, e.g., 'View notifications, 1 unread'. The profile dropdown toggle has an `aria-haspopup="true"` attribute.

Focus Management

A visible focus indicator (`:focus-visible`) is applied to all interactive elements, ensuring keyboard users can see where they are.

Aria Attributes

  • aria-label
  • aria-haspopup="true"
  • aria-expanded="false"

Color Contrast

Text and icon colors meet the WCAG 2.1 AA contrast ratio against the white background.
