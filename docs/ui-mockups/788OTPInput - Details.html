Description

A molecular component for entering One-Time Passwords (OTPs) or verification codes. It consists of a series of individual input fields that automatically handle focus shifts, backspacing, and pasting for a seamless user experience.

Design System Alignment

Utilizes the design system's form input styles for individual segments, including border radius, colors for default, focus, and error states, and spacing tokens for the gap between inputs. It adheres to the primary typography for numeric input.

Responsive Specifications

Mobile Adaptations

  • Input segments are slightly smaller with reduced gaps to fit on smaller screens.
  • Maintains large, touch-friendly tap targets for each input segment.

Tablet Adaptations

  • Maintains desktop sizing for consistency.

Desktop Enhancements

  • Full keyboard support including navigation and pasting.

Accessibility Features

Keyboard Support

Users can type digits, use backspace/delete, and paste codes. Focus moves automatically.

Screen Reader Support

Wrapped in a fieldset with a legend to announce the purpose of the input group. Each input has an `aria-label` specifying its position (e.g., 'Digit 1'). The error message is linked via `aria-describedby`.

Focus Management

Visible focus indicators are present. Focus automatically moves to the next input upon entry and the previous input upon backspace on an empty field.

Aria Attributes

  • aria-label
  • aria-describedby

Color Contrast

Input borders and text meet WCAG 2.1 AA contrast requirements in all states.
