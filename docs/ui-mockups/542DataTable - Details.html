Description

A responsive and accessible component for displaying sets of data in a tabular format. It supports states for loading, empty, and populated data, and includes dependencies like pagination and spinners. On smaller screens, it enables horizontal scrolling to handle wide tables gracefully.

Design System Alignment

The DataTable uses the design system's color palette for backgrounds, text, and borders. It utilizes the spacing scale for padding, typography for headers and cell content, and border-radius for the container. It also incorporates atomic components like Spinner and Button (for Pagination).

Responsive Specifications

Mobile Adaptations

  • The table wrapper enables horizontal scrolling to view all columns, preventing content from breaking the layout.
  • Pagination controls stack vertically for better usability on narrow screens.

Tablet Adaptations

  • Tablet devices will likely use the horizontal scroll behavior if the screen width is insufficient to display all columns comfortably.

Desktop Enhancements

  • The table is displayed in its entirety without horizontal scrolling on wider screens.
  • Row hover effects provide clear visual feedback for mouse users.

Accessibility Features

Keyboard Support

The table itself is not interactive, but interactive elements within cells (buttons, links) are focusable via Tab key. Pagination controls are fully keyboard navigable.

Screen Reader Support

Focus Management

Focusable elements within the table (and in pagination) have a clear `:focus-visible` state.

Aria Attributes

  • aria-sort on sortable table headers.
  • role="status" on a hidden element to announce loading state changes if implemented with JavaScript.
  • aria-label for pagination navigation buttons.

Color Contrast

Text and background colors are chosen from the design system to meet WCAG 2.1 AA contrast requirements.
