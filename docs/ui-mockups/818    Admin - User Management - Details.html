Responsive Specifications

Desktop Layout

  • Full-featured layout with a sidebar for navigation and a main content area.
  • Control bar with search and filters is displayed horizontally.
  • Data table shows all columns.

Tablet Adaptations

  • Sidebar may be collapsed by default.
  • Control bar elements may wrap if space is limited.
  • Data table becomes horizontally scrollable.

Mobile Adaptations

  • Sidebar is hidden behind a hamburger menu.
  • Control bar elements stack vertically.
  • Data table is horizontally scrollable to maintain readability of rows.

Accessibility Features

Keyboard Navigation

Logical tab order through all interactive elements: filters, search bar, sortable table headers, user action menus, and pagination.

Screen Reader Support

Semantic HTML (

, , etc.) is used. Table headers are scoped, and status badges include screen-reader-only text for context (e.g., 'Status: Active').

Focus Management

Visible focus states are applied to all interactive elements. Action menus manage focus trapping when open.

Aria Attributes

  • aria-label for icon buttons.
  • aria-sort for table headers.
  • aria-live for dynamic search results feedback.

Color Contrast

All text, UI controls, and status badges meet WCAG 2.1 AA contrast requirements.
