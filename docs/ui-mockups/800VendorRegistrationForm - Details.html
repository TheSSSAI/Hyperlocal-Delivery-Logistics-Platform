Description

A multi-step form for prospective vendors to register on the platform. It collects business details, contact information, and legal documents, guiding the user through the process with a clear step indicator.

Design System Alignment

This organism is composed of several molecular and atomic components from the design system, including FormGroup, Button, FileUpload, and Alert. It uses design system tokens for typography, color, spacing, and borders to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The horizontal step indicator transforms into a vertical layout to save horizontal space.
  • Form grids with multiple columns stack into a single column for better readability on narrow screens.

Tablet Adaptations

  • Maintains the desktop horizontal step indicator.
  • Form grid may reduce to a single column depending on viewport width.

Desktop Enhancements

  • Full horizontal step indicator is visible.
  • Multi-column form layouts are used to group related information and reduce vertical scrolling.

Accessibility Features

Keyboard Support

Focus Management

Focus is managed by standard browser behavior. When a new step is shown, focus should ideally be moved to the first input field of that step.

Aria Attributes

  • aria-live='polite' on the main form alert to announce submission errors.
  • aria-invalid='true' and aria-describedby for input fields in an error state.

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
