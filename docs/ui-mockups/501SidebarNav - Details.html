Description

A responsive sidebar navigation for web dashboards (Vendor, Admin). It's fully expanded on desktop, collapses to an icon-only view on tablets, and is hidden in an off-canvas menu on mobile.

Design System Alignment

This organism component is composed of atomic Link and Icon components. It heavily relies on the design system's tokens for color, typography, spacing, and transitions to create a cohesive and consistent navigation experience.

Responsive Specifications

Mobile Adaptations

  • Sidebar is hidden off-canvas by default.
  • A hamburger menu button (external to the component) is required to toggle its visibility.
  • When visible, it overlays the content with a backdrop.

Tablet Adaptations

  • Sidebar is permanently visible.
  • Collapses to an icon-only view by default (80px wide).
  • Expands on hover to show full text labels.

Desktop Enhancements

  • Sidebar is permanently visible and expanded by default (260px wide).
  • An optional toggle button can switch it to a collapsed state, which persists.

Accessibility Features

Keyboard Support

All navigation links are focusable with the Tab key and activatable with Enter. The toggle button is also keyboard accessible.

Screen Reader Support

Uses semantic `

` and `
` elements. Links have descriptive text. When collapsed, the link text still provides the accessible name, even though it's visually hidden.

Focus Management

Visible focus indicators are applied to all interactive elements.

Aria Attributes

  • aria-label for the main navigation element.
  • aria-current='page' for the active link.
  • aria-controls and aria-expanded for the toggle buttons.

Color Contrast

Text and icon colors meet WCAG 2.1 AA contrast requirements in all states (default, hover, active).
