Description

A specialized organism component for displaying interactive maps. It includes variants for live order tracking, address selection with a pin, and a geofencing tool for administrative use. The component is designed to be responsive and touch-friendly, with various states for loading and interactivity.

Design System Alignment

This component uses the design system's color palette, spacing, typography, and shadow tokens for its UI overlays (panels, buttons). It composes atomic components like Icon and Spinner for its functionality.

Responsive Specifications

Mobile Adaptations

  • Map aspect ratio changes to 1:1 to better fit vertical screens.
  • UI overlays and controls are designed with large touch targets (min 44x44px).
  • For the geofencing tool, the side panel transforms into a bottom sheet overlay to work on smaller screens.

Tablet Adaptations

  • Maintains the desktop layout but ensures that touch targets for controls remain large and easy to use.

Desktop Enhancements

  • Utilizes wider aspect ratios (16:9).
  • The geofencing tool variant uses a side-panel layout for controls and information, taking advantage of the available horizontal space.

Accessibility Features

Keyboard Support

All interactive UI overlay elements (buttons, form fields) are fully keyboard-navigable and operable.

Screen Reader Support

Buttons have descriptive `aria-label`s (e.g., 'Recenter map'). The map itself is a visual element; therefore, critical information like ETA or addresses displayed in info panels is accessible as standard text.

Focus Management

A visible focus outline is applied to all interactive controls, inherited from the atomic button styles.

Aria Attributes

  • aria-label for icon buttons
  • aria-busy='true' on the map container during loading state

Color Contrast

All text and interactive elements within the UI overlays meet WCAG 2.1 AA contrast requirements.
