Responsive Specifications

Desktop Layout

  • The Geofencing Tool variant is optimized for desktop use with a larger map area and mouse-precision for drawing.
  • Live tracking and address pickers are generally viewed within a constrained container.

Tablet Adaptations

  • Maintains the primary layout with larger touch targets for any interactive UI elements on the map.

Mobile Adaptations

  • Live tracking and address picker variants are designed mobile-first.
  • UI overlays (like ETA panels or buttons) are anchored to the top or bottom of the viewport for easy reachability.

Accessibility Features

Keyboard Navigation

Interactive elements on map overlays (buttons, toolbars) must be keyboard navigable.

Screen Reader Support

Informational overlays (like ETA) must be readable by screen readers. Map icons should have accessible text alternatives (e.g., 'Rider's current location').

Focus Management

Focus indicators must be visible on interactive overlay elements.

Aria Attributes

  • aria-label for icon buttons on overlays.
  • aria-busy='true' on the map container during the loading state.

Color Contrast

Text and interactive elements on map overlays must meet WCAG 2.1 AA contrast ratios against their immediate background.
