Description

A self-contained widget for admin dashboards, designed to display key information. It has variants for showing single Key Performance Indicators (KPIs), line charts for trends, and simple lists. It also includes states for loading, error, and displaying data.

Design System Alignment

This organism component is a composition of molecular (Card) and atomic (Spinner, Alert, Button, Icon) components. It strictly adheres to the design system's tokens for color, typography, spacing, and shadows to maintain visual consistency across the dashboard.

Responsive Specifications

Mobile Adaptations

  • Widgets are designed to stack vertically in a single column on mobile screens.
  • Font sizes for KPI values are slightly reduced to better fit smaller viewports.

Tablet Adaptations

  • Widgets maintain their desktop appearance but can be arranged in a 2-column grid.

Desktop Enhancements

  • Widgets are typically arranged in a multi-column grid (3 or 4 columns) to provide a comprehensive overview.

Accessibility Features

Keyboard Support

Interactive elements within the widget, such as dropdowns or links, are focusable via Tab key.

Screen Reader Support

Each widget is wrapped in a semantic `

` with an `aria-labelledby` attribute pointing to the widget's title (`

Focus Management

Focusable elements have a clear, visible focus state inherited from the atomic button/link components.

Aria Attributes

  • aria-labelledby
  • role='alert' for the error state content

Color Contrast

All text and data visualizations (e.g., chart colors) must meet WCAG 2.1 AA contrast requirements.
