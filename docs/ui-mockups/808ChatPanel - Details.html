Description

A real-time chat interface for communication between platform users (e.g., Customer, Vendor, Rider) within the context of an active order. It supports message history, text input, quick replies, and different states based on the order lifecycle.

Design System Alignment

This component is an organism that composes several smaller components: MessageBubble (molecular), TextInput (atomic), Button (atomic), and QuickReplyChip (molecular). It uses the design system's tokens for colors, spacing, typography, and shadows.

Responsive Specifications

Mobile Adaptations

  • The component is designed to take up the full screen height in a modal or dedicated screen.
  • The layout naturally adapts to narrower widths with flexbox.
  • Quick replies are horizontally scrollable to accommodate smaller screens.

Tablet Adaptations

  • Can be used in a docked sidebar or a larger modal view.

Desktop Enhancements

  • Primarily used in a 'docked' variant within a larger dashboard layout.
  • Hover states on quick replies and buttons provide better affordance.

Accessibility Features

Keyboard Support

The text input and send button are keyboard accessible. Quick replies are implemented as buttons and are also keyboard accessible.

Screen Reader Support

The message list should be in a container with an `aria-live` region so new messages are announced. The header provides context for the chat. Input fields have associated labels.

Focus Management

Focus is managed within the text input and send button. Opening the panel should focus the text input.

Aria Attributes

  • aria-live='polite' for the message container.
  • aria-label for icon-only buttons (e.g., 'Close chat').

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast ratios.
