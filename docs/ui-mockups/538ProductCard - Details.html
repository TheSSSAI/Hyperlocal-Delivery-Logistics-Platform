Description

A self-contained card component to display product information, including image, name, price, stock status, and actions. It has variants for grid and list layouts and adapts responsively.

Design System Alignment

Utilizes design system tokens for colors, spacing, typography, border-radius, and shadows. Composes atomic components like Badge, Button, and Icon.

Responsive Specifications

Mobile Adaptations

  • Defaults to 'list-view' which is a horizontal layout optimized for single-column scrolling.
  • Image has a 1:1 aspect ratio and fixed width to align with text.
  • Footer items (price, action) stack vertically to save horizontal space.

Tablet Adaptations

  • On screens wider than 768px, transitions to a multi-column grid layout.
  • Card changes to 'grid-view' (vertical) layout.
  • Image takes full width of the card with a 4:3 aspect ratio.

Desktop Enhancements

  • Same as tablet, utilizing a grid layout for product discovery.
  • Hover effects (lift and shadow) provide richer feedback on interactive cards.

Accessibility Features

Keyboard Support

The entire card is a focusable element (if it's a link). Action buttons within are also focusable. Tabbing follows a logical order.

Screen Reader Support

Card is wrapped in an `

` tag. The product name is an `

`. All interactive elements like buttons have descriptive `aria-label`s. Stock status is conveyed via text, not just color.

Focus Management

A visible focus outline appears on the card or its interactive elements when focused.

Aria Attributes

  • `aria-label` on buttons (e.g., 'Add Spicy Ramen to cart').
  • `aria-disabled='true'` is implicitly handled by the `disabled` attribute on the button for out-of-stock items.

Color Contrast

All text-to-background color combinations meet WCAG 2.1 AA requirements. Badge colors are selected for readability.
