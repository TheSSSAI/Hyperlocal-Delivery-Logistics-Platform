Description

A versatile component for displaying tabular data with support for sorting, filtering, selection, and various states like loading, empty, and error. It is designed to be responsive, providing horizontal scrolling on smaller viewports.

Design System Alignment

Uses design system tokens for colors, typography, spacing, and border-radius to ensure visual consistency. Interactive elements like checkboxes and buttons inherit their styles from the atomic component definitions.

Responsive Specifications

Mobile Adaptations

  • The table wrapper becomes horizontally scrollable on viewports smaller than 768px to prevent content from breaking the layout.
  • Padding within cells is maintained to ensure readability.
  • The primary adaptation is scrolling, not transformation to cards, to maintain the data relationships for complex datasets.

Tablet Adaptations

  • Depending on the number of columns, the table may be fully visible or require horizontal scrolling.

Desktop Enhancements

  • The table is expected to be fully visible without horizontal scrolling in most common use cases.
  • Hover states on rows provide clear visual feedback.

Accessibility Features

Keyboard Support

Sortable headers are focusable and can be activated with Enter/Space. Checkboxes in rows are part of the natural tab order.
