Description

A versatile container for displaying grouped content and actions. Cards are used to present distinct pieces of information such as vendor profiles, products, order summaries, and key performance indicators (KPIs).

Design System Alignment

The card component is a foundational molecular component that uses core design system tokens for spacing, typography, color, border-radius, and shadow to ensure a consistent look and feel across the platform.

Responsive Specifications

Mobile Adaptations

  • ProductCard variant stacks its image and content vertically on screens < 768px.
  • Main showcase grid changes to a single-column layout for better readability on small screens.

Tablet Adaptations

  • Cards maintain their desktop layout on tablets.

Desktop Enhancements

  • Hover effects provide visual feedback.
  • ProductCard variant uses a horizontal layout to efficiently use space.

Accessibility Features

Keyboard Support

Cards are focusable via Tab key. Interactive elements within cards follow a logical tab order.

Screen Reader Support

Using semantic elements like

for cards and headings (h3, h4) for titles provides a clear structure for screen reader users.

Focus Management

A clear, high-contrast focus outline is applied to the card when it receives keyboard focus.

Aria Attributes

N/A for the base card, but interactive elements within (like buttons) should have appropriate ARIA labels if their content is not descriptive.

Color Contrast

All text within the cards meets WCAG 2.1 AA contrast ratio requirements against their background.
