Description

A navigation component used to traverse through a series of pages or a large set of content. It provides controls to go to the previous, next, or a specific page number, and adapts for smaller screens.

Design System Alignment

This component utilizes the design system's tokens for colors (primary, neutrals), spacing, border-radius, and typography to ensure visual consistency with other components like Buttons and Icons.

Responsive Specifications

Mobile Adaptations

  • The component simplifies to 'Previous' and 'Next' buttons with an informational text like 'Page X of Y'.
  • The full list of page numbers is hidden to save space.

Tablet Adaptations

  • The full component is displayed, including page numbers and ellipsis.
  • Maintains touch-friendly target sizes for all interactive elements.

Desktop Enhancements

  • Full component view with clear hover and focus states for all interactive links and buttons.

Accessibility Features

Keyboard Support

All links and buttons are focusable and activatable using Tab and Enter/Space keys.

Screen Reader Support

The component is wrapped in a

element with an `aria-label="pagination"` for clear navigation landmarking. The current page is marked with `aria-current="page"`. Disabled buttons use `disabled` attribute.

Focus Management

A clear, high-contrast focus ring is applied to all interactive elements on `:focus-visible`.

Aria Attributes

  • aria-label="pagination"
  • aria-current="page"

Color Contrast

Text and background colors for default, current, and focus states meet WCAG 2.1 AA contrast requirements.
