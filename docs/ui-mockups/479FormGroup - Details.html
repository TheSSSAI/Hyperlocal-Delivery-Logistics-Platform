Component Id

Component Name

FormGroup

Description

A fundamental form component that groups a label, an input field, and optional help text or an error message. It ensures proper spacing, accessibility, and state management for a single form entry.

Component Type

molecular

Design System Alignment

Combines the atomic 'Label' and 'TextInput' components, and uses 'Alert' styling for error states. It utilizes the design system's typography, spacing, and color tokens for a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The component stacks vertically by default, making it inherently mobile-friendly.
  • Ensures full width usage within its container for optimal use of screen real estate.

Tablet Adaptations

  • Maintains vertical stacking layout.

Desktop Enhancements

  • Can be placed within multi-column grid layouts for more complex forms.

Accessibility Features

Keyboard Support

The input element within the group is focusable via the Tab key.

Screen Reader Support

The

Focus Management

The input element has a clear, high-contrast focus ring when active, defined by the --shadow-focus token.

Aria Attributes

  • aria-describedby is used to link help and error text to the input.
  • aria-invalid='true' is applied to the input in the error state.
  • The error message container can have role='alert' in a JS implementation to announce errors.

Color Contrast

Label, input text, help text, and error text all meet WCAG 2.1 AA contrast ratios.
