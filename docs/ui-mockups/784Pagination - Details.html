Description

Provides navigation between pages of a list, table, or any paginated content. It includes controls for previous/next pages and direct navigation to specific page numbers. The component is responsive, simplifying on mobile devices.

Design System Alignment

This component utilizes the design system's primary and neutral color palettes for states, spacing tokens for padding and layout, typography for text, and border-radius for shape consistency.

Responsive Specifications

Mobile Adaptations

  • Displays a simplified view with only 'Previous' and 'Next' buttons to save space.
  • Maintains large, touch-friendly 44x44px targets for all interactive elements.

Tablet Adaptations

  • Expands to the full view, showing page numbers and ellipsis for truncation.

Desktop Enhancements

  • Full view with hover effects for better user feedback.

Accessibility Features

Keyboard Support

Component is fully navigable using the Tab key. Focused items have a visible focus state. Enter/Space keys activate the buttons.

Screen Reader Support

The component is wrapped in a `

` element with `aria-label="pagination"`. The current page is identified with `aria-current="page"`. Disabled buttons use `aria-disabled="true"`.

Focus Management

A clear, high-contrast focus outline is applied using `:focus-visible` to avoid clutter for mouse users while ensuring keyboard navigability.

Aria Attributes

  • aria-label="pagination"
  • aria-current="page"
  • aria-disabled="true"

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast ratios against their backgrounds in all states.
