Description

A navigation component for paginated content. It allows users to navigate through a series of pages, showing the current page and links to nearby pages. It adapts for mobile by simplifying the interface.

Design System Alignment

Combines atomic Button and Icon components. Utilizes the design system's tokens for color, spacing, typography, and borders to ensure a consistent navigation experience.

Responsive Specifications

Mobile Adaptations

  • Hides numerical page links and ellipsis.
  • Displays a compact view with only 'Previous' and 'Next' arrows and a text indicator like 'Page 2 of 10'.
  • Maintains large 44x44px touch targets for arrows.

Tablet Adaptations

  • Displays the full desktop version with all page numbers.

Desktop Enhancements

  • Shows a full set of page numbers, including ellipsis for larger page ranges.
  • Includes hover effects for interactive elements.

Accessibility Features

Keyboard Support

All interactive elements (arrows, page numbers) are focusable with Tab and activatable with Enter/Space.

Screen Reader Support

The main container is a `

` with an `aria-label`. The current page is marked with `aria-current="page"`. Disabled arrows have `aria-disabled="true"`.

Focus Management

A clear, visible focus indicator is applied to all interactive elements on `:focus-visible`.

Aria Attributes

  • aria-label="pagination"
  • aria-label="Previous page"
  • aria-label="Next page"
  • aria-current="page"
  • aria-disabled="true"

Color Contrast

Text and icons meet WCAG 2.1 AA contrast requirements against their backgrounds in all states.
