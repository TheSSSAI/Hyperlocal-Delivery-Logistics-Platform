Description

A highly specialized component for displaying interactive maps. It has variants for different use cases: live order tracking for customers, an address picker with a pin for location selection, and a polygon drawing tool for administrators to define geofenced zones.

Design System Alignment

The MapView component uses atomic components like 'Icon' and 'Spinner' for its overlays and controls. All UI elements on top of the map (buttons, panels, text) adhere to the design system's tokens for color, typography, spacing, and shadows to ensure a consistent user experience.

Responsive Specifications

Mobile Adaptations

  • The map's aspect ratio changes to be more portrait-oriented (4:5) to better fit mobile screens.
  • Overlay panels have reduced horizontal margins to maximize map visibility.
  • All interactive elements like buttons and toolbar icons maintain large, touch-friendly targets.

Tablet Adaptations

  • Maintains the desktop aspect ratio (16:9).
  • UI panels and controls are sized appropriately for touch interactions.

Desktop Enhancements

  • Wider aspect ratio (16:9) is optimal for desktop viewing.
  • The Geofencing Tool variant is primarily designed for desktop use with mouse precision for drawing polygons.

Accessibility Features

Keyboard Support

Interactive elements within the map overlays (buttons, inputs) are keyboard-focusable.

Screen Reader Support

Since the map itself is a visual component, screen reader support focuses on the overlay content. Live tracking panels announce status and ETA. Form panels have properly labeled inputs.

Focus Management

When a panel or modal appears over the map, focus should be programmatically moved to that element.

Aria Attributes

  • aria-label for icon-only buttons in toolbars.
  • aria-busy='true' on the map container during the loading state.

Color Contrast

All text and interactive elements in the UI overlays meet WCAG 2.1 AA contrast requirements.
