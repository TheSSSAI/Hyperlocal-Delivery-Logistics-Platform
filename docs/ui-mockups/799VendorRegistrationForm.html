Component Variations

Variant Name : "multi-step"

The form is presented as a series of logical steps to guide the user through the data collection process without overwhelming them. This is the primary and only variant of this component. The different states below demonstrate its lifecycle.

Variant Name : "Submitting State"

After the user clicks the final 'Submit' button, the form is overlaid with a loading indicator to provide feedback that the submission is in progress. The form content is not interactive during this time.

Variant Name : "Success State"

Upon successful submission, the loading indicator is replaced with a success icon and a confirmation message. This provides a clear end to the registration journey.

Variant Name : "Error State"

If form validation fails on any step, the respective form fields are highlighted, and an inline error message appears. This state is managed at the molecular `FormGroup` level. A form-wide error (e.g., API failure on submit) would be shown using the `Alert` component.
