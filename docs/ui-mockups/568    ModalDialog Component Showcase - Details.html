Responsive Specifications

Desktop Layout

  • The modal is vertically and horizontally centered within the viewport.
  • It has a maximum width and is surrounded by a semi-transparent overlay.
  • Maintains a card-like appearance with box-shadow.

Tablet Adaptations

  • Maintains the desktop layout but may have slightly larger touch targets for buttons.

Mobile Adaptations

  • The modal expands to fill the entire screen or a majority of it.
  • The overlay is not visible, but the modal still traps focus.
  • Border radius is reduced or removed for a flush, full-screen appearance.

Accessibility Features

Keyboard Navigation

When a modal is open, focus is trapped within it. The user can only tab between the interactive elements inside the modal. Pressing 'Escape' closes the modal.

Screen Reader Support

The modal container uses `role="dialog"` and `aria-modal="true"`. It is labeled by its heading using `aria-labelledby`.

Focus Management

When a modal opens, focus is programmatically moved to the first interactive element (e.g., the close button or the first form input). When it closes, focus returns to the element that opened it.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and interactive elements within the modal adhere to WCAG 2.1 AA contrast requirements.
