Responsive Specifications

Desktop Layout

  • Sidebar is in its 'expanded' state by default, showing icons and text labels.
  • A toggle allows the user to switch to the 'collapsed' state.

Tablet Adaptations

  • Sidebar defaults to the 'collapsed' (icon-only) state to maximize content area.
  • The toggle allows the user to expand it if needed.

Mobile Adaptations

  • Sidebar is hidden off-screen by default.
  • A hamburger icon in the main header toggles its visibility as an off-canvas overlay.
  • When open, it covers part of the screen and a scrim covers the remaining content.

Accessibility Features

Keyboard Navigation

The sidebar is fully navigable using the Tab key. The navigation list follows a logical tab order.

Screen Reader Support

The main component is wrapped in a `

` element with an `aria-label`. Each link has descriptive text, and icons are decorative (`aria-hidden='true'`). The collapse/expand button has an `aria-label` describing its function.

Focus Management

Visible focus states are applied to all interactive elements. On mobile, focus is trapped within the off-canvas menu when it is open.

Aria Attributes

  • aria-label
  • aria-current='page' for the active link
  • aria-hidden='true' for decorative icons

Color Contrast

All text and icon colors meet WCAG 2.1 AA contrast ratios against the background.
