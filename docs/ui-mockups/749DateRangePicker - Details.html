Description

A form control that allows users to select a start and end date from a calendar interface, often including predefined date ranges. It is composed of a text input trigger and a popover containing calendar views and preset options.

Design System Alignment

This component utilizes the design system's color palette for states (primary for selection, neutrals for background), spacing scale for padding and margins, typography for text, and border-radius for a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The popover stacks vertically: presets on top, followed by a single calendar month.
  • Popover width is constrained to fit mobile viewports.
  • Preset buttons may wrap into multiple lines.

Tablet Adaptations

  • At a specific breakpoint (e.g., > 900px), the layout switches to a horizontal arrangement with presets on the left and two calendar months side-by-side on the right.

Desktop Enhancements

  • The default view shows two calendar months for easier range selection across month boundaries.
  • Utilizes hover effects for interactive elements like preset buttons and calendar days.

Accessibility Features

Keyboard Support

The trigger is a standard button. The popover should trap focus. Calendar days are buttons, navigable with arrow keys. Presets and footer buttons are navigable with Tab.

Screen Reader Support

The trigger button announces its purpose. The popover should have `role="dialog"` and `aria-modal="true"`. Each calendar day button should have an `aria-label` with the full date (e.g., 'January 1, 2024'). Selected dates should announce their state (e.g., 'start date').

Focus Management

Focus is moved into the popover upon opening and returned to the trigger on close. Visible focus states are applied to all interactive elements.

Aria Attributes

  • aria-label
  • aria-modal
  • role="dialog"

Color Contrast

Selected states use high-contrast combinations (primary color on white text) that meet WCAG AA standards.
