Description

A responsive sidebar navigation component for web dashboards. It provides primary navigation links with icons and text. It supports expanded and collapsed states on desktop/tablet and an off-canvas overlay state on mobile.

Design System Alignment

This component utilizes the design system's color palette (primary for active states, neutrals for background and text), typography scales for link text, and spacing tokens for padding and layout. It is designed to contain atomic Link and Icon components.

Responsive Specifications

Mobile Adaptations

  • Sidebar is hidden off-canvas by default.
  • A hamburger menu icon appears to toggle the sidebar.
  • When open, the sidebar overlays the content with a dark backdrop.
  • The collapse/expand toggle in the footer is hidden.

Tablet Adaptations

  • The sidebar is permanently visible and forced into the 'collapsed' (icon-only) state.
  • Main content area has a left margin equal to the collapsed sidebar's width.

Desktop Enhancements

  • Sidebar is permanently visible and defaults to the 'expanded' state.
  • A toggle button in the footer allows the user to switch between 'expanded' and 'collapsed' states.
  • Hover effects on navigation links provide clear interactive feedback.

Accessibility Features

Keyboard Support

All navigation links and the toggle button are focusable and can be activated with Enter/Space. The tab order is logical.

Screen Reader Support

The main component is wrapped in a

element with `aria-label="Main navigation"`. The active link is marked with `aria-current="page"`. The toggle button has an `aria-expanded` attribute that is updated via JavaScript.

Focus Management

Visible focus states are applied to all interactive elements. On mobile, when the off-canvas menu is opened, focus should be trapped within it.

Aria Attributes

  • aria-label
  • aria-current
  • aria-expanded
  • aria-controls

Color Contrast

All text has a minimum contrast ratio of 4.5:1 against the dark background, meeting WCAG 2.1 AA requirements.
