Description

A blocking overlay and container for displaying focused content, such as confirmations, forms, or critical alerts. It requires a user action to be dismissed and traps focus to meet accessibility standards.

Design System Alignment

Utilizes the design system's color palette for backgrounds and semantic actions, spacing scale for padding, typography for content, and incorporates atomic components like Button, Icon, and TextInput.

Responsive Specifications

Mobile Adaptations

  • Modal dialog takes up the full screen height and width to maximize space.
  • Scrolling is handled within the modal body.

Tablet Adaptations

  • The modal appears as a centered dialog with a backdrop, similar to the desktop view, but with generous padding around it.

Desktop Enhancements

  • The modal is a centered dialog with a fixed max-width.
  • An overlay dims the background content.
  • Entry and exit animations provide a smooth user experience.

Accessibility Features

Keyboard Support

The `Esc` key closes the modal. `Tab` key navigation is trapped within the modal content.

Screen Reader Support

The dialog uses `role="dialog"` and `aria-modal="true"`. The title is linked via `aria-labelledby` and descriptive text is linked via `aria-describedby`.

Focus Management

When the modal opens, focus is moved to the first focusable element inside it. When it closes, focus is returned to the element that triggered it.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet WCAG 2.1 AA contrast requirements.
