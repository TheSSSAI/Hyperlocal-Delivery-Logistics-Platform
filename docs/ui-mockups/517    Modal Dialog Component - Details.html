Responsive Specifications

Desktop Layout

  • Modal is centered horizontally and vertically within the viewport.
  • Modal has a maximum width to prevent it from becoming too wide on large screens.
  • A semi-transparent backdrop covers the entire viewport.

Tablet Adaptations

  • Maintains the desktop layout with increased tap target sizes for buttons.

Mobile Adaptations

  • Modal transitions to a full-screen or near-full-screen view.
  • Modal is anchored to the top or bottom of the screen for easier reachability.
  • Header and footer remain fixed while the body content is scrollable if it overflows.

Accessibility Features

Keyboard Navigation

Focus is trapped within the open modal. Users can only tab through interactive elements inside the modal.

Screen Reader Support

Uses `role='dialog'`, `aria-modal='true'`, `aria-labelledby`, and `aria-describedby` to provide context to screen readers.

Focus Management

When a modal opens, focus is programmatically moved to the first focusable element inside it. When it closes, focus returns to the element that triggered it.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text, icons, and buttons within the modal adhere to WCAG 2.1 AA contrast requirements.
