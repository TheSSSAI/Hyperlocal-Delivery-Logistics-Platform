Description

A vertical navigation component for web dashboards. It provides primary navigation links with icons and text, and is designed to be responsive by collapsing to an icon-only state on tablets and hiding in an off-canvas menu on mobile.

Design System Alignment

This component is an organism built from molecular and atomic components like 'Link' and 'Icon'. It strictly adheres to the design system's tokens for color, typography, spacing, and transitions, ensuring a cohesive look and feel within the Vendor and Admin dashboards.

Responsive Specifications

Mobile Adaptations

  • The entire sidebar is hidden off-screen to the left (`transform: translateX(-100%)`).
  • When toggled (e.g., by a hamburger button in the AppHeader), it slides into view and an overlay (`backdrop`) covers the main content.
  • It maintains its expanded width when open on mobile to ensure link text is readable.

Tablet Adaptations

  • On screens between 769px and 1024px, the sidebar defaults to the 'collapsed' (icon-only) state to maximize content area.
  • The toggle button to expand/collapse is hidden, as this state is enforced by the viewport width.

Desktop Enhancements

  • The sidebar is 'expanded' by default, showing both icons and text for clear navigation.
  • A toggle button allows the user to switch between 'expanded' and 'collapsed' states to customize their view.
  • Hover states on navigation links provide clear feedback.

Accessibility Features

Keyboard Support

All navigation links and the toggle button are focusable and can be activated using Enter or Space keys.

Screen Reader Support

The component is wrapped in a `

` landmark. Each link provides full text, which is ideal for screen readers. In the collapsed state, links should ideally use `aria-label` to announce the full destination if text is visually hidden.

Focus Management

A clear `:focus-visible` state is applied to all interactive elements, using the design system's primary color outline.

Aria Attributes

  • `aria-current="page"` is used on the link for the currently active page.
  • `aria-expanded` and `aria-controls` are used on the toggle buttons to indicate state and control relationships.

Color Contrast

Text and icon colors have a minimum 4.5:1 contrast ratio against the dark background, meeting WCAG 2.1 AA requirements.
