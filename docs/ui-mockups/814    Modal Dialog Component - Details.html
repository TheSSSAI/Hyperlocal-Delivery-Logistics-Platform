Responsive Specifications

Desktop Layout

  • Modal is centered vertically and horizontally.
  • Modal has a max-width (e.g., 500px) and vertical margins.
  • The background page content is visible but inert under a semi-transparent overlay.

Tablet Adaptations

  • Maintain desktop layout with slightly larger touch-friendly targets within the modal.

Mobile Adaptations

  • Modal takes up a larger portion of the screen width (e.g., 90vw).
  • For very small screens, the modal may appear near the top or bottom of the screen to avoid the virtual keyboard.
  • Spacing and font sizes remain consistent, but the overall container adapts.

Accessibility Features

Keyboard Navigation

When a modal is open, focus is trapped within it. Users can tab through interactive elements inside the modal. The 'Escape' key closes the modal.

Screen Reader Support

The modal container has `role='dialog'`, `aria-modal='true'`, and is properly labeled with `aria-labelledby` pointing to the modal title.

Focus Management

When a modal opens, focus is programmatically moved to the first focusable element inside it (e.g., a close button or a form field). When it closes, focus returns to the element that triggered it.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and interactive elements within the modal adhere to WCAG 2.1 AA contrast requirements.
