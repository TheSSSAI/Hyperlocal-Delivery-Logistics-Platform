Description

A self-contained card for displaying a specific piece of data on a dashboard. It can be configured to show a key performance indicator (KPI), a chart visualization, or a simple list. It supports loading, error, and data-rich states.

Design System Alignment

This component is built upon the atomic 'Card' primitive, inheriting its padding, border-radius, and shadow from the design system. It uses typography tokens for titles and values, and semantic color tokens for trend indicators and error states. It also composes atomic Spinner and Alert components for its loading and error states.

Responsive Specifications

Mobile Adaptations

  • Widgets are designed to be fluid and will take the full width of their container. On a mobile screen, the parent grid will stack them into a single vertical column.

Tablet Adaptations

  • The parent grid might display widgets in a two-column layout.

Desktop Enhancements

  • Widgets are typically arranged in a multi-column grid (e.g., 3 or 4 columns) on desktop dashboards, allowing for a high-level overview.

Accessibility Features

Keyboard Support

The component itself is not interactive, but if it contains links or buttons (e.g., in a list-widget), those elements are keyboard-focusable.

Screen Reader Support

The widget is contained within a `

` with an `aria-labelledby` attribute pointing to the widget's title. This provides a clear landmark for screen reader users. In the loading state, `aria-busy='true'` is applied. In the error state, the message has an appropriate ARIA role.

Focus Management

N/A as the base widget is not interactive.

Aria Attributes

  • aria-busy

Color Contrast

All text and semantic colors are sourced from the design system to meet WCAG 2.1 AA contrast requirements.
