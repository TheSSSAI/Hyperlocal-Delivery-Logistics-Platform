Description

A FormGroup is a layout component that groups a label, a form input (like TextInput or Textarea), and an optional error message. It manages the vertical spacing and accessibility connections between these elements.

Design System Alignment

This component utilizes the design system's spacing tokens for gaps and margins, typography for labels and error messages, and semantic colors for error states.

CSS Implementation

/* FormGroup Component CSS */ /* Base styles for the component */ .form-group { display: flex; flex-direction: column; gap: var(--space-8); margin-bottom: var(--space-16); } /* Error message is hidden by default */ .form-group .form-error-message { display: none; } /* When the FormGroup has an error, display the error message */ .form-group.has-error .form-error-message { display: block; } /* The form-input within an error group gets the error styling */ .form-group.has-error .form-input { border-color: var(--color-semantic-error); } /* Focus state on an error input should maintain the error color */ .form-group.has-error .form-input:focus, .form-group.has-error .form-input.is-focus { border-color: var(--color-semantic-error); outline: 3px solid var(--color-danger-50); }

Implementation Notes

Embedded Css Architecture

The FormGroup CSS is designed to work in conjunction with the atomic styles for Label, TextInput, and ErrorMessage. It primarily handles layout and the conditional display of the error message.

Component Organization

The HTML must correctly link the label's 'for' attribute to the input's 'id', and the input's 'aria-describedby' to the error message's 'id'.

Responsive Design

The component is inherently responsive due to its simple, vertical stacking layout.

Accessibility Compliance

Proper use of 'id', 'for', 'aria-describedby', and 'aria-invalid' is critical for achieving WCAG 2.1 AA compliance.

Browser Support

Uses standard CSS Flexbox, supported by all modern browsers.

Usage Guidelines

When To Use

Use this component as the standard wrapper for any single form input that requires a label and may show a validation error. It provides consistent spacing and accessibility.

Content Guidelines

The label should be concise and descriptive. The error message should be clear and actionable.
