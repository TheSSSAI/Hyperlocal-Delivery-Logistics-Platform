Description

A navigation component for paginated content, allowing users to move between pages of a list. It is responsive, showing full controls on desktop and simplified controls on mobile.

Design System Alignment

Utilizes design system tokens for colors (primary, neutrals), spacing, typography, and border-radius to ensure visual consistency with atomic components like Buttons and Links.

Responsive Specifications

Mobile Adaptations

  • Simplified view displays only 'Previous' and 'Next' buttons for a compact interface.
  • Page number links and ellipsis are hidden to save space.
  • Maintains a 44px touch target for accessibility.

Tablet Adaptations

  • Starting from the 768px breakpoint, the full pagination component is displayed.
  • Page number links and ellipsis separators become visible.

Desktop Enhancements

  • Full pagination component is visible with all controls.
  • Hover effects provide clear visual feedback on interactive elements.

Accessibility Features

Keyboard Support

All interactive elements (buttons, links) are focusable and can be activated using Tab, Enter, and Space keys.

Screen Reader Support

The component is wrapped in a

element with an `aria-label` of 'pagination'. The current page is marked with `aria-current='page'`. 'Previous' and 'Next' buttons have descriptive `aria-label` attributes.

Focus Management

A clear, high-contrast focus indicator is applied to all interactive elements on focus.

Aria Attributes

  • aria-label="pagination"
  • aria-label="Go to previous page"
  • aria-label="Go to next page"
  • aria-label="Go to page X"
  • aria-current="page"

Color Contrast

Text and background colors meet WCAG 2.1 AA contrast requirements. Active and disabled states are visually distinct.
